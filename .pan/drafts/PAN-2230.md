## 0. Glossary (define before use)

- **Import cycle** — a chain of ES-module imports that returns to its start
  (`A imports B imports A`). Node.js strict ESM rejects some require/import cycles at
  runtime ("Cannot require() ES Module … in a cycle"); Bun and bundlers tolerate them.
  This is why the dashboard can only run from the tsdown-bundled `dist/dashboard/server.js`
  and not from source under Node — the standing dev-loop tax this ratchet stops from growing.
- **madge** — the dependency-graph CLI this PRD adds (`madge@^8`, dev dependency).
  `madge --circular --json` emits a JSON array of cycles, each an array of file paths
  relative to the analyzed directory. **Verified on this repo:** it correctly resolves the
  house ESM import style (`from './foo.js'` specifiers pointing at `.ts` sources) — a full
  `src/` scan finds 116 cycles in ~7 s, including 6 in the frontend.
- **Canonical cycle line** — the baseline's stable text form of one cycle: rotate the
  member list so the lexicographically smallest path is first (a cycle has no natural
  start, and madge's start point varies), prefix each member with `src/` (madge emits
  src-relative paths), join with ` > `. Example:
  `src/lib/agents.ts > src/lib/agents/activity.ts`.
- **Baseline** — `scripts/circular-deps-baseline.txt` (new): all current canonical cycle
  lines, sorted, deduplicated. Shrink-only: removals free, additions audited (issue
  reference in the adding commit, enforced via PAN-2227's ratchet audit once registered).
- **Barrel re-export cycle** — the deliberate pattern where a decomposed god file
  re-exports from its submodules while they import readers back from it (e.g.
  `dashboard/server/routes/workspaces.ts > dashboard/server/routes/workspaces/merge-ops.ts`).
  These are tolerated-by-bundler cycles, present in today's 116, and are baselined like any
  other — the ratchet does not judge cycle kinds, it stops NET NEW ones.
- **Guard-script convention** — bash script, `#!/usr/bin/env bash`, `set -euo pipefail`,
  `cd "$(dirname "$0")/.."`, wired into `npm run lint` via a `lint:*` entry (the style of
  `scripts/lint-file-size.sh`).

## 2. Locked design decisions

- **D1 — madge, not dependency-cruiser.** Verified live at authoring: `madge@8` with
  `--ts-config tsconfig.json` resolves this repo's `.js`-suffixed ESM imports and finds
  cycles in `src/lib`, `src/dashboard/server`, AND `src/dashboard/frontend` (6 frontend
  cycles), full scan ~7 s. dependency-cruiser is rejected: it brings a config file and
  rules DSL when the repo's established ratchet pattern is a bash guard + text baseline.
  This decision is already verified — do NOT re-litigate it; if madge's output shape
  changed in a newer major, pin to `^8`.
- **D2 — madge is a pinned devDependency invoked from `node_modules/.bin`.** Add with
  `bun add -d madge` (Bun is the package manager; `bun.lock` changes are expected in the
  diff). The guard runs `"${MADGE_BIN:-node_modules/.bin/madge}"` — never `npx madge`
  (network-dependent, unpinned). The `MADGE_BIN` env override is the test seam (WI-4).
- **D3 — Exit-code handling.** `madge --circular` exits **1 when any cycle exists**
  (verified) — which is always, given the baseline. The guard must capture the JSON with
  the exit code tolerated (`out=$(... || true)` shape) and fail ONLY on baseline
  comparison, never on madge's own exit code. A madge invocation error (empty/unparsable
  output) must fail loudly — distinguish it from "no cycles" (`[]`).
- **D4 — Exact canonical cycle lines, not a count.** A count would let a new cycle hide
  behind a removed one. Cost accepted: a decomposition that reshapes a cycle's membership
  produces one stale line + one new line, requiring `--update` plus an issue reference in
  the commit — which a decomposition PR has by construction.
- **D5 — Shrink-only + `--update`, mirroring PAN-2227/PAN-2231 semantics.** Check mode
  fails on (a) a cycle not in the baseline (names the canonical line, tells the author to
  break the cycle or — if consciously accepted — add the line in a commit referencing an
  issue) and (b) a stale baseline line (cycle no longer exists) with the fix command
  `bash scripts/lint-circular-deps.sh --update`. `--update` drops stale lines, never adds.
  A `--print` mode emits the current canonical lines (used once for WI-3's initial
  baseline generation and by anyone auditing).
- **D6 — Register the baseline with PAN-2227's ratchet audit** as a plain-text set file
  (added line = increase requiring an issue reference; removals free). **Implementation
  checkpoint:** if `scripts/lint-ratchet-audit.sh` does not exist at execution (PAN-2227
  not landed), skip, leave a `TODO(PAN-2230)` in the WI-1 script header, and report the
  gap — do not implement a parallel audit.
- **D7 — Known undercount is accepted and documented.** madge silently skips imports it
  cannot resolve (path aliases outside the root tsconfig, dynamic imports). An
  unresolvable edge can hide a cycle. This is a v1 limitation noted in the script header;
  tightening resolution (e.g. frontend `@/` aliases via the frontend tsconfig) is
  follow-up, not this issue.

## 4. Requirements

### Functional
- **FR-1** A new guard `scripts/lint-circular-deps.sh` fails (exit 1) when the current
  `src/` import graph contains a cycle whose canonical line is not in
  `scripts/circular-deps-baseline.txt`, naming each new line and both remedies (break the
  cycle; or consciously baseline it in a commit referencing an issue).
- **FR-2** Check mode fails on stale baseline lines (cycle gone) with the
  `--update` fix command; `--update` drops stale lines, never adds; `--print` emits the
  current canonical lines sorted.
- **FR-3** Canonicalization is rotation-invariant: the same cycle reported from any start
  member produces the identical canonical line (smallest-member-first rotation, `src/`
  prefix, ` > ` join).
- **FR-4** madge's exit code 1 (cycles exist) is tolerated; an empty or unparsable madge
  output fails loudly with a distinct message (D3).
- **FR-5** The guard is wired into `npm run lint` as `lint:circular-deps`; `madge@^8` is a
  root devDependency installed via Bun.
- **FR-6** The initial baseline is generated with `--print` at execution time, committed in
  the same change, and the guard passes on the current tree.
- **FR-7** (checkpoint, D6) the baseline is registered in `scripts/lint-ratchet-audit.sh`
  with set semantics, or the fallback is executed and reported.

### Non-functional
- **NFR-1** Guard completes in ~10 s or less on this repo (verified ~7 s) — acceptable for
  the lint chain (which already runs builds via `prelint:skills`).
- **NFR-2** Deterministic: sorted, deduplicated canonical lines; stable across runs on an
  unchanged tree (verify by running twice and diffing).
- **NFR-3** No production `src/**` changes; no cycle is broken in this issue.
- **NFR-4** New files stay under 1,000 lines (file-size guard); the canonicalizer is an
  inline `node -e` in the guard script (no new production module).

## 5. Work items

### WI-1 — `scripts/lint-circular-deps.sh` *(FR-1..4)*

**Files:** new `scripts/lint-circular-deps.sh`.

**Steps:**
1. Guard-script convention header; document D4 (exact lines), D5 (`--update` never adds),
   D7 (undercount limitation), and the audited-addition path.
2. Run madge per D2/D3:
   ```bash
   MADGE_BIN="${MADGE_BIN:-node_modules/.bin/madge}"
   json=$("$MADGE_BIN" --circular --json --extensions ts,tsx --ts-config tsconfig.json src/ || true)
   ```
3. Canonicalize with an inline `node -e` reading the JSON from stdin: parse (parse failure
   or non-array → print `✖ madge produced no usable output` to stderr, exit 2 — FR-4);
   for each cycle array, rotate so the lexicographically smallest member is index 0, map
   members to `src/<member>`, join with ` > `; print unique lines sorted.
4. Modes: `--print` → emit lines, exit 0. `--update` → baseline := (baseline ∩ current
   lines), sorted; print `✓ baseline updated: N dropped, M kept`. Default check → new lines
   (current ∖ baseline) each reported as
   `✖ new import cycle: <line>` with the two remedies; stale lines (baseline ∖ current) as
   `✖ stale baseline line (cycle gone): <line> — run: bash scripts/lint-circular-deps.sh --update`;
   any of either → exit 1; else
   `✓ circular-deps guard passed (<K> known cycles, none new)`.

`MADGE_BIN=<repo>/node_modules/.bin/madge` (resolve the repo root from
`import.meta.dirname` — madge is a real devDependency after WI-2, so the binary exists).
`execFileSync('bash', [script], { cwd: root, env: {...} })` — fine in vitest fixtures; the
no-`execSync` rule covers server-reachable production code. Cases:
1. `src/a.ts` importing `./b.js` and `src/b.ts` importing `./a.js` (house ESM style),

2. baseline containing that canonical line → passes with the `K known cycles` summary;
3. rotation invariance (FR-3): a 3-file cycle `c→a→b→c` produces a line starting with the
   lexicographically smallest member; assert the exact line;
4. stale line (acyclic fixture, non-empty baseline) → fails with the `--update`
   instruction; after `--update` the file is empty and check passes;
5. `--update` never adds: cycle present + empty baseline → still fails after `--update`;
6. FR-4 loud failure: `MADGE_BIN=/bin/true` (produces no output) → exit 2 with the
   `no usable output` message, NOT a pass.

No timers/retries/delays — the fake-timers rule (§6) applies to none of these tests.
