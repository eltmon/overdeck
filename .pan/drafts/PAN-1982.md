## Goal

Quick review (the single `agent-<id>-review` parent reviews the diff itself) stays the
**default**. Make the **full convoy review** (4 sub-reviewers + synthesis) selectable as an
opt-in at **three scopes â€” global, per-project, and per-issue** â€” so heavier review is
available on demand without slowing every issue.

## Glossary

- **Quick review** â€” one `agent-<id>-review` parent reviews the whole diff; no sub-reviewers,
  no synthesis. Current live default (`buildSelfReviewPrompt`).
- **Convoy / full review** â€” the 4 `REVIEW_SUB_ROLES` (security, correctness, performance,
  requirements) each review the diff, then the parent synthesizes a verdict
  (`buildReviewRolePrompt`). Currently PARKED (commented out).
- **Review mode** â€” `quick | full`: which of the above runs for a given review.
- **Resolution order** â€” per-issue â†’ project â†’ global â†’ default(`quick`). The first scope that
  sets a mode wins.
- **Per-issue record** â€” the permanent per-issue JSON at `.pan/records/<issue-lowercase>.json`
  (`PanIssueRecord`), read with `readIssueRecordSync` and written with `writeIssueRecordSync`
  (the single read/write door for that domain). Git-committed and portable.
