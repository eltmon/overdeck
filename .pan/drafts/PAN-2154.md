# PAN-2154 — Decompose `src/lib/workspace-manager.ts` into focused modules

**Type:** Refactor · codebase-health. **Behavior-preserving — pure relocation + re-exports.**

> This draft mirrors the executable plan in `.pan/spec.vbrief.json` (items `wi-0`…`wi-6`)
> and the decisions/hazards in `.pan/continue.json`. The vBRIEF is the source of truth.

## Glossary

- **Barrel** — `src/lib/workspace-manager.ts` after this refactor: no domain logic, only `import`s
  from sibling submodules, the two Effect-error helpers, the 9 Effect wrappers, and re-export
  statements that keep the public surface byte-identical so all importers resolve unchanged.
- **Submodule** — a new file under the **existing** `src/lib/workspace/` directory (already home to
  `devcontainer-renderer.ts`, `ensure-devcontainer.ts`, `rebuild-stack.ts`, `stack-health.ts`).
- **`*Sync` / `*Promise` function** — the synchronous impl / the private `async function` the barrel
  wraps via `Effect.try` / `Effect.tryPromise`.
- **No-loss audit gate** — `wi-0`: a frozen export-surface snapshot test that fails if any public
  named export disappears. Every later seam must keep it green (mirrors the PAN-2146 `agents.ts` pattern).
- **A1 ratchet** — `eslint-any-allowlist.json`: no NEW explicit `any`; a file legitimately carrying
  moved `any` must be listed.
- **File-size guard** — `scripts/lint-file-size.sh`, `CEILING=1000`; `workspace-manager.ts` is baselined
  at `1736` in `scripts/file-size-baseline.txt:46`.

## Problem

`src/lib/workspace-manager.ts` is **1736 lines** — over the 1000-line ceiling (grandfathered in the
baseline). It mixes five concerns plus an Effect-wrapper layer: pre-PAN-967 settings/migration sync
helpers, shared worktree/port/symlink/template primitives, workspace creation, repo-addition, and
workspace removal + Docker cleanup. 14 runtime symbols and 9 interfaces are exported; 14 source files
and 5 test files import from it, so the public surface must not change.

## Goal

Extract the five concerns into cohesive submodules under `src/lib/workspace/`, each under 1000 lines,
and reduce `src/lib/workspace-manager.ts` to a thin barrel that re-exports the **identical** public
surface. Pure moves + re-exports — no logic edits, no renamed/removed exports, **zero call-site or
test edits**.

`create.ts`/`add-repos.ts`/`remove.ts` import the shared primitives from `./worktree-helpers.js`;
`createWorktree` is shared by create + add-repos, `removeWorktree`/`releasePort` by remove — which is
why the primitives are a single shared module (decision D6). Submodules `export` their internal helpers
and `*Promise` functions so siblings and the barrel can import them, but the barrel re-exports **only**
the previously-public surface (relocateVenvScripts, preTrustDirectorySync, the `*Sync` fns, the 9 types,
the 9 Effect wrappers) — never an internal helper or a `*Promise` (hazard H3).

## Public surface that must stay byte-identical (wi-0 snapshot — 14 runtime value exports)

`migrateOverdeckToPanSync`, `copyOverdeckSettingsToWorkspaceSync`, `ensurePanGitignoreSync`,
`relocateVenvScripts`, `preTrustDirectorySync`, `migrateOverdeckToPan`, `copyOverdeckSettingsToWorkspace`,

`getContainersReferencingWorkspacePath`, `stopWorkspaceDocker`, `removeWorkspace`.
(Interfaces erase at runtime, so they are not in the snapshot; `export type { … }` re-exports keep them
resolving for typecheck.)

## Resolution-safety note

A file `src/lib/workspace-manager.ts` and the directory `src/lib/workspace/` coexist cleanly:
`import './workspace-manager.js'` resolves to the barrel file; submodules import each other as
`./worktree-helpers.js` etc. (ESM `.js`-extension convention used throughout `src/lib`, hazard H5).

## Method — one seam per commit, gates after each

Sequence (see `edges`): `wi-0` → { `wi-1`, `wi-2` in parallel } ; `wi-1` → { `wi-3`, `wi-4`, `wi-5` } ;
all → `wi-6`. `wi-3`/`wi-4`/`wi-5` each edit the shared barrel, so serialize the barrel edits even
though their submodules are independent. Per seam: move verbatim, add the new file to
`eslint-any-allowlist.json` if it carries existing `any` (all but `worktree-helpers.ts`), keep the wi-0
snapshot green, then run the gates and commit `refactor(workspace-manager): extract <seam>`.

### Gate commands (after EVERY seam — all 0 / 0 failed)

```bash
npm run typecheck
npm run lint            # includes lint:file-size and the eslint any-guard
npx vitest run --configLoader runner
```

## Requirements

- **FR-1** Every public symbol still resolves from `src/lib/workspace-manager.ts`; the wi-0 snapshot
  reports 0 differences at every seam. Zero edits to the 14 importers or 5 test files.
- **FR-2** Each new `src/lib/workspace/*.ts` file is `< 1000` lines; the barrel is materially smaller (~150).
- **FR-3** Behavior-preserving: pure moves + re-exports. No logic edits, no renamed/removed exports.
  `createSymlinks` (dead code, hazard H1) moves verbatim — never deleted.
- **NFR-1** A1 ratchet: no NEW explicit `any`; each new file carrying moved `any` is listed in
  `eslint-any-allowlist.json` in the same commit. The allowlist has no unused-entry check, so the
  barrel's stale entry is harmless (decision D7) — optionally removed in wi-6 only if lint stays green.
- **NFR-2** Async only — preserve `execAsync`; introduce no `execSync`.
- **NFR-3** File-size guard green; wi-6 lowers the `1736 src/lib/workspace-manager.ts` baseline line to
  the new barrel size so the ceiling can't silently re-inflate.

## Acceptance criteria

1. `wi-0` snapshot test exists and reports 0 differences against the final barrel.
2. Five submodules under `src/lib/workspace/` exist, each `< 1000` lines; the barrel is a thin entry (~150 lines).
3. `npm run typecheck` exits 0; `npm run lint` exits 0 (file-size + any guards green);
   `npx vitest run --configLoader runner` reports 0 failed.
4. `git diff --name-only main` shows only the barrel, the new `src/lib/workspace/*.ts`, the wi-0 test,
   `eslint-any-allowlist.json`, and `scripts/file-size-baseline.txt` (and `.pan/` planning artifacts) —
   no importer or pre-existing test file changed.
5. Each commit's `git diff` is a pure move + wiring.

## Intersecting repo rules (restated)

No bandaids; surgical moves only. Worktree discipline — stay on `feature/pan-2154`, never
`git checkout <branch>`/`git stash`. Conventional commits, never `--no-verify`. Server code is
Node-22 / async-only. This IS a normal pipeline issue: `pan done` when all seams are green → review → merge.

## Out of scope

Any logic/signature/export change; deleting dead `createSymlinks`; modifying the existing
`src/lib/workspace/` modules beyond importing from them; the dashboard `routes/workspaces.ts`
decomposition (separate effort, `B-workspaces-decomp.md`).
