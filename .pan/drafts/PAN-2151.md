# PAN-2151 — Decompose `src/dashboard/server/routes/misc.ts` into focused submodules

**Issue:** [PAN-2151](https://github.com/eltmon/overdeck/issues/2151)
**Type:** Refactor — codebase-health (`substrate-improvement`). Behavior-preserving.
**Campaign:** `docs/codebase-health/REMAINING-BACKLOG.md` (A3 file-size guard).

---

## Glossary

- **Barrel** — a module whose only job is to re-export symbols defined elsewhere, so existing importers resolve unchanged. Here, `misc.ts` becomes the barrel.
- **Seam** — one cohesive extraction unit: a group of related routes (plus the helpers only they use) moved to a new file in a single commit, gated independently.
- **Route layer** — an Effect `Layer` produced by `HttpRouter.add(...)`. Each route in `misc.ts` is a layer; `miscRouteLayer` merges them all via `Layer.mergeAll(...)`.
- **Sub-layer** — the merged `Layer.mergeAll(...)` of one submodule's routes, exported from that submodule and merged into `miscRouteLayer` by the barrel.
- **File-size guard** — `scripts/lint-file-size.sh` (wired into `npm run lint`). Ceiling is 1000 lines for any new non-test `src` file; a *baselined* file (listed in `scripts/file-size-baseline.txt`) may shrink but never grow.
- **A1 ratchet** — `npm run lint` forbids new explicit `any`. Files that legitimately carry `any` are listed in `eslint-any-allowlist.json`.
- **Cross-group helper** — a helper used by routes that land in **two or more** different submodules. These move to the shared module. A helper used by only one submodule's routes moves into that submodule.

## Problem

`src/dashboard/server/routes/misc.ts` is **1832 lines** (baselined at 1835 in `scripts/file-size-baseline.txt:20`). It is a catch-all for ~37 API route handlers spanning unrelated concerns: trackers, project-mappings, system/agent health, deacon control, planning sessions, TLDR daemon, cache, metrics, shadow monitoring, and dev/restart endpoints. The file is a god-file flagged by the A3 file-size campaign.

The public surface is tiny — only **two exports**:

- `readPackageVersion()` — `misc.ts:92` (consumed by `tests/dashboard/version-api.test.ts:13`)
- `miscRouteLayer` (`misc.ts:1791`) + `export default miscRouteLayer` (`misc.ts:1832`) — consumed by `src/dashboard/server/server.ts:51`

Everything else is module-private: ~12 helper functions/constants (lines 92–322) and ~37 route definitions (lines 324–1789) merged into `miscRouteLayer`.

## Proposal

Pure relocation. Split the route handlers into cohesive **topical submodules** under a new sibling directory `src/dashboard/server/routes/misc/`, each exporting a sub-layer. Reduce `misc.ts` to a **thin barrel** that:

1. Re-exports `readPackageVersion` (keeps `version-api.test.ts` and any importer resolving unchanged — **zero test edits**).
2. Composes `miscRouteLayer = Layer.mergeAll(<all sub-layers>, bootReconciliationRouteLayer)` and `export default miscRouteLayer` (keeps `server.ts` resolving unchanged).

**No logic edits. No renamed exports. No new public API.** Helpers move with their routes; helpers used across groups move to a shared module.

### Why a sibling `misc/` directory (not flat files in `routes/`)

Keeps the relocation self-evidently scoped to "the old misc.ts contents" and avoids polluting `routes/` with a dozen new top-level files. `boot-reconciliation.ts` (already a sibling at `routes/boot-reconciliation.ts`, imported at `misc.ts:86`) stays where it is — it is already extracted; the barrel keeps merging `bootReconciliationRouteLayer` exactly as today.

## Decisions made in this document

- **Decision — barrel keeps `readPackageVersion`.** `readPackageVersion` moves into the shared module and is **re-exported** from `misc.ts`. This makes `tests/dashboard/version-api.test.ts:13` (`import { readPackageVersion } from '../../src/dashboard/server/routes/misc.js'`) resolve unchanged, so no test is repointed. (The issue's "repoint any test importing moved internals" clause is satisfied by *not moving the import target* — the barrel still exports it.)
- **Decision — cross-group helper rule.** A helper used by routes ending up in **2+** submodules → goes in `misc/shared.ts`. A helper used by only one submodule's routes → moves into that submodule. This rule is mechanical; apply it, do not deliberate per helper. Verified cross-group helpers (by usage grep): `readJsonBody` (8 call sites across project-mappings, rally, deacon, confirmations, planning, shadow — `misc.ts:314`), `getProjectPath` (planning + shadow — `misc.ts:167`) and its dependencies `isGitHubIssue` (`misc.ts:181`) + `getGitHubLocalPaths` (`misc.ts:194`). Verified group-local helpers: `getProjectMappings`/`saveProjectMappings`/`PROJECT_MAPPINGS_FILE`/`ProjectMapping` (project-mappings only — `misc.ts:141–166`), `pendingConfirmations`/`ConfirmationRequest` (confirmations only, **no external writers** — `misc.ts:209,218`), `checkPlanStatus`/`PLANNING_FINISHED_STATUSES` (planning only — `misc.ts:220,222`), `loadRuntimeMetrics`/`METRICS_FILE` (metrics only — `misc.ts:237,239`), `getIndexStats` (tldr only — `misc.ts:250`), `getIssueDataService` (cache-status only — `misc.ts:134`). `overdeckDevMode` (`misc.ts:119`, 3 uses) and `getOverdeckVersion`/`_overdeckVersion` (`misc.ts:110,111`) → apply the rule (likely shared with version/dev routes; if used by only one submodule, move there).
- **Decision — one seam per commit, gate between each.** Each submodule extraction is its own commit, each followed by the full gate (below). This matches the issue's "After each seam" requirement and keeps any regression bisectable to one move.
- **Decision — seams serialize.** Every seam edits `misc.ts` (removes moved code; adds the new import + sub-layer to the composition). That is shared mutation of one file, so seams run **sequentially**, not in parallel. The vBRIEF edges encode a linear chain for this reason — do not parallelize across subagents.
- **Decision — the shared module lands first.** `misc/shared.ts` is the foundation every other seam imports from, so it is item 1.

## Submodule layout (target)

```
src/dashboard/server/routes/
  misc.ts                  # BARREL: re-export readPackageVersion; compose + default-export miscRouteLayer
  boot-reconciliation.ts   # unchanged (already extracted)
  misc/
    shared.ts              # readPackageVersion, readJsonBody, getProjectPath, isGitHubIssue,
                           #   getGitHubLocalPaths (+ getOverdeckVersion/overdeckDevMode if cross-group)
    trackers.ts            # trackersRouteLayer  (refresh, tracker-status, rally validate)
    project-mappings.ts    # projectMappingsRouteLayer (+ mappings helpers)
    health.ts              # healthRouteLayer (system-health, godview, health/agents, ping)
```

Each submodule pattern:

```ts
// misc/health.ts
import { Layer } from 'effect';
import { HttpRouter, ... } from 'effect/unstable/http';
// ...moved imports it actually uses...

// ...other health routes moved verbatim...

  getSystemHealthRoute,
  getGodviewSystemHealthRoute,

  postHealthAgentPingRoute,

```

Barrel after all seams:

```ts
// misc.ts (barrel)
import { Layer } from 'effect';
import { bootReconciliationRouteLayer } from './boot-reconciliation.js';
import { trackersRouteLayer } from './misc/trackers.js';
import { projectMappingsRouteLayer } from './misc/project-mappings.js';
import { healthRouteLayer } from './misc/health.js';
import { deaconRouteLayer } from './misc/deacon.js';
import { planningRouteLayer } from './misc/planning.js';
import { tldrRouteLayer } from './misc/tldr.js';
import { metaRouteLayer } from './misc/meta.js';

export { readPackageVersion } from './misc/shared.js';

export const miscRouteLayer = Layer.mergeAll(
  trackersRouteLayer,
  projectMappingsRouteLayer,

  metaRouteLayer,

export default miscRouteLayer;
```

> Note: `Layer.mergeAll` is order-independent for route registration — preserving the original ordering is not required for correctness, but keep a sensible order for readability.
