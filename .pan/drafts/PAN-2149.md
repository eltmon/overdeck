## 0. Glossary

- **`service.ts`** — `src/lib/cloister/service.ts` (2,057 lines): the Cloister lifecycle
  watchdog service. Two regions: ~580 lines of module-level **reactive dispatch** free
  functions (domain-event → role spawn), and the ~1,350-line **`CloisterService` class**
  (grep anchor: `export class CloisterService {`) owning the monitoring loop, crash
  handling, completion markers, alerts, and status surface.
- **Barrel decomposition** — the B3/agents pattern (`docs/codebase-health/B3-agents-queries-extract.md`,
  `docs/codebase-health/refactor-case-study.md`): move code verbatim into
  `cloister/service-<seam>.ts`, then `export { … } from './service-<seam>.js'` in
  `service.ts` so all 14+ importers (verified: `main.ts`, `routes/cloister.ts`,
  `routes/misc.ts`, `routes/metrics.ts`, `routes/specialists.ts`, `routes/agents.ts`,
  `cli/commands/kill.ts`, `cli/commands/cloister/*`, `overdeck/process-services.ts`, …)
  resolve unchanged.
- **Host-object delegation** — the pattern for thinning CLASS methods (a pure move can't
  relocate a method that touches `private` members, and TS structural typing rejects an
  interface that demands access to another class's privates). The extracted function
  lives in the seam module and takes a `host` parameter; the class method becomes a
  delegate that builds the host **object literal at the call site**, closing over `this`
  for private members. See D2 for the exact before/after.
- **Verification gate (PAN-174)** — the quality-gate flow triggered from completion
  markers; `checkCompletionMarkers` is its entry inside this file. Highest-care seam.
- **Reactive dispatch** — `handleCloisterDomainEventPromise` and its helpers: on an issue
  state change, spawn the matching role run. The `agent.stopped`/fast-path code
  (PAN-2203's insertion point) lives here too if that epic has landed (§8.1).

## 2. Locked design decisions

- **D1 — Barrel, not thinning.** All existing exports keep resolving from
  `src/lib/cloister/service.ts`; zero importer edits. (Contrast Batch 2: routes got
  thinned with repointing because a route file is transport; `service.ts` is a lib module
  whose import surface is legitimately wide.)
- **D2 — Class methods extract via host-object delegation.** Template:

  **Before** (inside `CloisterService`):
  ```ts
  private async performHealthCheck(): Promise<void> {
    // ~120 lines using this.config, this.agentHealth, this.emit(...), …
  }
  ```
  **After** — in `service-health.ts`:
  ```ts
  export interface HealthCheckHost {
    config: CloisterConfig;
    agentHealth: Map<string, AgentHealth>;
    emit: (event: CloisterEvent) => void;
    // …exactly the members the body uses — enumerate while moving, nothing more
  }
  export async function performHealthCheck(host: HealthCheckHost): Promise<void> {
    // body moved verbatim; `this.` → `host.`
  }
  ```
  and in the class:
  ```ts
  private async performHealthCheck(): Promise<void> {
    return performHealthCheck({
      config: this.config,
      agentHealth: this.agentHealth,
      emit: (event) => this.emit(event),
    });
  }
  ```
  Private members cross the boundary ONLY as host fields/closures built at the call
  site — never widen a `private` to `public` for extraction, and never pass raw `this`.
- **D3 — Module-level free functions move verbatim** (no host needed) — the reactive
  seam is a pure move + re-export, same as B3.
- **D4 — Seam sizing targets** (verify with `wc -l` as you go): `service-reactive.ts`
  ≈450, `service-health.ts` ≈350, `service-crash.ts` ≈300, `service-completion.ts`
  ≈150; `service.ts` ends ≈800 (must be <1,000 and its baseline entry lowered).
  If a seam lands >1,000, split it further before committing — never bump a baseline up.
- **D5 — One seam per commit; FULL `npx vitest run --configLoader runner` (0 failed) per
  seam** — not just the touched tests. This is the file the deacon and the review
  pipeline run through; TENET-10 makes partial-suite confidence insufficient.
- **D6 — Cycle discipline.** `service.ts` re-exporting from `service-<seam>.ts` while
  seams import types back from `service.ts` creates bundler-tolerated cycles (the B3
  precedent). `npm run build` success is the real check; if PAN-2230's ratchet is live,
  run `bash scripts/lint-circular-deps.sh --update` per seam — new cycle lines are
  audited additions carried by this issue's ref. If a RUNTIME ESM cycle error appears in
  the build, stop and report — do not paper over with logic changes.
