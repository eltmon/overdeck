Contextual action resolver — what surfaces, when

HealthyOpen UAT ↗   API ↗  ·  (Restart in overflow). Chip = green.
StartingLogs (watch boot)  ·  spinner. Chip = amber. No destructive actions while transient.
UnhealthyLogs (primary — diagnose)   Rebuild. Chip = red "N/M unhealthy" + the first failure reason inline.
Stopped (idle)Start (primary — bring back up)   Reap. Chip = gray "stopped 4d" — not red. This is the screenshot case.
Stale (merged)Reap workspace (primary). Chip = gray "merged · idle 4d". The issue's done; stop showing it as a health problem.

The action set is derived from (stack state × issue lifecycle) — the same "surface the right operator action for the context" principle generalizes to other error rows (red main, troubled agent, orphaned resources).
