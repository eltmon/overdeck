# PAN-1894 — UAT environment in the issue tree: collapsible tree-child, honest status, contextual actions

**Status:** Ready · **Type:** substrate-improvement (dashboard UX)
**Visual spec:** [`.pan/drafts/PAN-1894-mockup.html`](PAN-1894-mockup.html) — open in a browser; flip the *Stack scenario* buttons and right-click the "UAT environment" row. The mockup is the source of truth for layout, status colors, and the action set. Build to match it.

---

## Glossary

- **Feature row** — a single issue row in the Command Deck project tree, rendered by `FeatureItem` (`src/dashboard/frontend/src/components/CommandDeck/ProjectTree/FeatureItem.tsx`).
- **Tree child** — an indented row *under* a feature row, drawn with the tree's left guide line and chevron (e.g. sessions, `ReviewGroup`, `ResourcesGroup`). Visually part of the tree, not a floating card.
- **UAT stack** — the per-workspace Docker compose stack (postgres/redis/api/frontend/dev/init/test) that the workspace boots so browser UAT can run against it. Health/containers come from `useWorkspaceQuery` → `WorkspaceContainerStatus` / `WorkspaceStackHealth`.
- **Paper card** — the current rendering: a self-bordered `rounded-md border bg-muted/20` box that floats inside the tree instead of aligning as a tree child.
- **Contextual action resolver** — a pure function mapping `(stack state × issue lifecycle)` to the 1–2 operator actions most appropriate right now (shown inline) plus the full action set (shown in the right-click menu).
- **Honest status** — distinguishing `stopped` (cleanly stopped / idle-reaped — *not a failure*) from `unhealthy` (containers crashed) from `starting`. Today a cleanly-stopped stack is shown red as "0/7 healthy".

## Problem (verified against current code)

1. **Not collapsible.** `FeatureItem.tsx:1284-1288` renders a `<ChevronDown size={12}>` that is purely decorative — no state, no `onClick`. `shouldShowUatStack` (`FeatureItem.tsx:1028` = `expanded && feature.readyForMerge && hasWorkspace`) forces the block open whenever the feature is expanded. The operator cannot collapse it.
2. **Renders as a "paper card", not a tree child.** `UatStackStatus` (`UatStackStatus.tsx:108`, markup at `:127-171`) wraps everything in `<div className="rounded-md border border-border bg-muted/20 p-2.5">`, and each container is another bordered box (`:143`). Inside the `uatStackTreeGroup` (CSS at `command-deck.module.css:5867`, which already adds `border-top` + `padding-left:28px`) this double-borders and floats like a pasted sheet, misaligned from sibling tree children (sessions, `ResourcesGroup`).
3. **No actions for error conditions.** `UatStackStatus` only surfaces Frontend/API links, and only when `stackHealth.healthy === true` (`:133-137`, `:156-168`). When the stack is unhealthy or stopped there is **no** Rebuild / Restart / Start / Reap / Logs control anywhere.
4. **Status semantics conflate stopped with unhealthy.** `normalizeStatus` (`UatStackStatus.tsx:16-25`) maps a cleanly-stopped container to a red/failed presentation, and `getUatStackSummary` (`:82-106`) reports "N/M healthy" with the warning spinner. A stack that was cleanly `compose down`/idle-reaped (exit 0/255 from SIGTERM) shows identically to a crash. This is the [PAN-1740](https://github.com/eltmon/overdeck/issues/1740) "SIGTERM ≠ crash" problem surfacing in the tree.

## Requirements

- **FR-1** The UAT section is a real collapsible tree child: a chevron toggle with persisted expand state, **collapsed by default**.
- **FR-2** The UAT row and its containers render in the tree's row style (indent guide, no self-border card). No "paper".
- **FR-3** Status is honest: `healthy` (green) / `starting` (amber) / `unhealthy` (red) / `stopped` (gray) / `stale` (gray). A cleanly-stopped stack is never shown red.
- **FR-4** The 1–2 most appropriate operator actions for the current `(stack state × issue lifecycle)` render inline on the row and are **always visible** (not hover-gated).
- **FR-5** Right-clicking the UAT row opens a context menu with the full available action set (primary actions on top, destructive `Reap` at the bottom), reusing the existing `ContextMenu` primitives.
- **NFR-1** Behavior-preserving for the data path: same `useWorkspaceQuery` source; no new polling. Frontend tests + typecheck + lint green.

## Design (build to the mockup)

The mockup `.pan/drafts/PAN-1894-mockup.html` shows the target for every scenario. Key decisions, already made:

- **Row = status.** The collapsed row carries a colored status chip (`7/7 healthy` green · `starting…` amber · `2/7 unhealthy` red · `stopped 4d` gray · `merged · idle 4d` gray). The first failure reason renders inline under the row for `unhealthy` (amber) and as a neutral note for `stopped`/`stale` (muted, not amber).
- **Expand → container tree rows.** Each container is a plain tree row: small status dot + name + status text (`Up 2h` / `exited (255)` / `exited (0) 4d ago`). No bordered card.
- **Contextual action resolver** (the mockup's `SCEN[*].actions` + `buildMenu`):

  | Stack state | Inline (always visible) | Right-click adds |
  | --- | --- | --- |
  | healthy | Open **UAT** ↗, **API** ↗ | View logs, Restart, Rebuild, Stop, Open state dir, Copy stack name, Reap |
  | starting | **Logs** | Open UAT/API, Restart, Rebuild, Stop, … |
  | unhealthy | **Logs** (primary), **Rebuild** | Open UAT/API, Restart, Stop, Open state dir, Copy stack name, Reap |
  | stopped (idle) | **Start** (primary), **Reap** | View logs, Rebuild, Open state dir, Copy stack name |
  | stale (merged) | **Reap workspace** (primary) | View logs, Start, Open state dir, Copy stack name |

  The resolver is a pure function of `(stackState, issueLifecycle)` and is the reusable core — the same shape applies later to other error rows (troubled agent, orphaned resources). Keep it a standalone exported function so it is unit-testable.

Do not block the UI work on the backend — ship the frontend with this heuristic and tighten when PAN-1740 lands.

## Acceptance criteria (map 1:1 to FRs)

- **AC-1 (FR-1):** The UAT row is collapsed by default; clicking the chevron toggles it; the state persists across reloads (localStorage key distinct from the feature's). A test asserts default-collapsed + toggle.
- **AC-2 (FR-2):** No `rounded-md border`/`bg-muted` card in the UAT tree rendering; containers render as indented tree rows aligned with sibling children. Snapshot/DOM test asserts the wrapper class is gone in `density="tree"`.
- **AC-3 (FR-3):** A fully cleanly-stopped stack renders a **gray** "stopped" chip (no red, no spinner); an `unhealthy` stack renders red with the failure reason. Unit test on `resolveUatStatus`/`getUatStackSummary` for the stopped-vs-unhealthy-vs-starting matrix.
- **AC-4 (FR-4):** For each stack state, the documented inline actions are present and visible **while expanded** (regression guard for the hover bug). Unit test on `resolveUatActions`.
- **AC-5 (FR-5):** Right-clicking the UAT row opens a menu containing the full action set incl. `Reap` as destructive. Test asserts the menu items for at least the `unhealthy` and `stopped` states.

## Intersecting repo rules (restated for the implementer)

- **Dashboard build:** after changing server code run `npm run build`; the dashboard runs `dist/dashboard/server.js` under Node 22 — never Bun.
- **No `execSync`** in server-reachable code — use `execAsync`/`spawn`.
- **Quality gates before `pan done`:** `npm run typecheck`, `npm run lint`, `npm test` (root + frontend) all green.
- **Reap = existing teardown primitive only.** Never deep-wipe, never a new destructive path; destructive actions require explicit confirm.
- Surgical changes — touch only the UAT rendering path; don't refactor adjacent tree code.

## Out of scope

- Generalizing the action resolver to non-UAT error rows (troubled agents, orphaned resources) — the resolver is built standalone so that is a later, easy follow-up, but not part of this issue.
