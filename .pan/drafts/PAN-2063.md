# PAN-2063 — Collapse UAT stack by default in dashboard

**Issue:** https://github.com/eltmon/overdeck/issues/2063
**Planned:** 2026-06-26 (auto mode, claude-opus-4-8)

## Problem

The "UAT environment" panel inside an expanded feature row currently always shows
its full per-service list, adding visual clutter to a surface that is rarely the
focus during active development. Worse, when the stack is unhealthy the panel is
*forced* fully expanded, so the noisiest case (`0/7 healthy`) is also the loudest.

## Glossary

- **Feature row** — one issue card in the Command Deck project tree, rendered by
  `FeatureItem.tsx`. It has its own outer expand/collapse (the row caret).
- **UAT panel / UAT stack group** — the "UAT environment" block that renders inside
  an *expanded* feature row when the issue is ready-for-merge and has a workspace.
  Markup: `styles.uatStackTreeGroup` (header `styles.uatStackTreeHeader` + body
  `<UatStackStatus density="compact" />`).
- **`feature-uat-stack` badge** — a *separate* one-line status pill in the row's
  badge cluster (`FeatureItem.tsx:1203`, "UAT starting" / "UAT healthy"). This is
  NOT the panel and is out of scope — it stays visible.

## Verified references (against main @ d5ab0c3ad)

- **UAT panel render site** — `src/dashboard/frontend/src/components/CommandDeck/ProjectTree/FeatureItem.tsx:1282-1298`.
  Header (`uatStackTreeHeader`) draws a `<ChevronDown size={12} />` that is purely
  decorative today — no toggle state is wired, so the chevron implies a
  collapsibility that does not exist. Body is `<UatStackStatus density="compact" />`.
- **Header summary** — `FeatureItem.tsx:1287` renders
  `uatStackSummary.label.replace(/^UAT stack\s*/i, '')` → e.g. `"0/7 healthy"`.
- **Why it always expands** — `UatStackStatus.tsx:122`:
  `const showDetails = density === 'full' || pending || stackHealth?.healthy === false;`
  When unhealthy, `showDetails` is forced `true`, so the per-service list
  (`UatStackStatus.tsx:140-155`) always renders. This is the screenshot case.
- **Both chevrons already imported** — `FeatureItem.tsx:5` imports `ChevronRight`
  and `ChevronDown`; no new import needed.
- **Row-caret pattern to mirror** — `FeatureItem.tsx:1080-1086`: a `<button>` with
  `aria-label`, `title`, and `{expanded ? <ChevronDown/> : <ChevronRight/>}`.
- **CSS** — `styles/command-deck.module.css:5867-5891` (`.uatStackTreeGroup`,
  `.uatStackTreeHeader`, `.uatStackTreeSummary`). Header is a muted, uppercase,
  letter-spaced flex `<div>` today; converting it to a `<button>` needs a button
  reset (appearance/background/border none, width 100%, inherit font, cursor
  pointer) while preserving the existing muted style per `pan-style-guide`.
- **Existing test that breaks** — `FeatureItem.test.tsx:626`
  ("shows expanded UAT environment state for ready-for-merge issues") asserts the
  *expanded* body by default: `findByText('UAT stack 1/2 healthy')`,
  `getByText('api unhealthy: connection refused')`, `getByText('postgres')`,
  `getByText('api')` — all of which come from inside `UatStackStatus` and will be
  hidden once the panel defaults to collapsed. `getByTestId('feature-uat-stack')`
  and `findByText('UAT environment')` stay valid. This test MUST be updated in the
  same change to keep the vitest suite green.

## Proposal (recommended in the issue, verified feasible)

Make the **wrapper** (`FeatureItem`) own collapse state for the UAT panel and
default it to collapsed; keep `UatStackStatus` a pure presenter (do not touch
`UatStackStatus.tsx`).

1. Add `const [uatExpanded, setUatExpanded] = useState(false);` to `FeatureItem`.
2. Convert `uatStackTreeHeader` from a `<div>` to a `<button>` that toggles
   `uatExpanded`, exposes `aria-expanded`, and renders
   `{uatExpanded ? <ChevronDown size={12}/> : <ChevronRight size={12}/>}`. The
   "UAT environment" label and the `uatStackTreeSummary` span stay in the header so
   health is glanceable while collapsed.
3. Render `<UatStackStatus .../>` only when `uatExpanded`.
4. Add a button-reset to `.uatStackTreeHeader` (and a hover color) so it looks the
   same but behaves as a toggle.
5. Update `FeatureItem.test.tsx:626` to assert collapsed-by-default (header +
   summary present, service rows absent) and click-to-expand (service rows appear).

`UatStackStatus.tsx:122` (`showDetails`) is left unchanged: it only governs what
renders *inside* `UatStackStatus` once the wrapper chooses to mount it (expanded),
which is the desired behavior. The workspace-health query at `FeatureItem.tsx:1028-1030`
(`enabled: shouldShowUatStack`) is also left unchanged — it must keep running while
the panel body is collapsed so the header summary count stays populated.

## Planning decisions (auto mode — no interactive Q&A)

- **D1 — Wrapper owns collapse, presenter stays pure.** Chosen over editing
  `UatStackStatus.tsx`'s `showDetails`, because the issue's recommended fix is to
  gate the whole panel at the wrapper and the existing `feature-uat-stack` badge +
  header summary already give a glanceable health signal when collapsed.
- **D2 — Session-local state only; NO localStorage persistence.** The issue flags
  persistence as explicitly *optional* ("if we want the collapse to stick"). The
  literal requirement is "collapsed by default"; a clutter-reduction feature is best
  served by every fresh render starting collapsed. The existing `readExpanded`/
  `writeExpanded` helpers are tied to the row-expand key (`mc-feature-expanded:`),
  so reusing them would collide with row state; persistence would need a *new* key
  and helpers — added surface for an optional behavior. Deferred to NonGoals.
  (If the operator later wants stickiness, add a `mc-uat-expanded:<issueId>` key and
  mirror the row-expand persistence pattern.)
- **D3 — Single bead.** Implementation + CSS + the existing-test update are one
  cohesive, independently-mergeable change. Splitting the test into its own bead
  would leave bead 1 shipping a RED suite (the existing UAT test breaks), violating
  the "independently mergeable / tree stays working" rule.

## Acceptance

- On a ready-for-merge feature row with an unhealthy UAT stack, the UAT panel
  renders collapsed by default: the "UAT environment" header and its `N/M healthy`
  summary are visible; the per-service rows (e.g. `postgres`, `api`) are not.
- Clicking the header expands the panel (renders `UatStackStatus` and its service
  rows); clicking again collapses it.
- The header is a button with `aria-expanded` and a chevron reflecting state
  (ChevronRight collapsed, ChevronDown expanded).
- `FeatureItem.test.tsx` covers collapsed-by-default and click-to-expand; the
  frontend vitest suite is green; `npm run typecheck` and `npm run lint` pass.

## NonGoals

- No localStorage persistence of the UAT collapse state (see D2).
- No change to `UatStackStatus.tsx` (`showDetails`, summary, density behavior).
- No change to the `feature-uat-stack` badge or the outer feature-row caret.
- No change to the workspace-health polling/query gating.
