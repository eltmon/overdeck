## 1. Problem (verified evidence at `506756378f`)

Production no longer imports the cluster (verified sweep; the single exception is
PAN-1983's backlog re-export). But 31 files under `tests/` plus 10 under
`src/lib/database/__tests__/` still bind to it via real imports, so:

1. The 18 modules (≈4,500 lines incl. the 1,650-line baselined `schema.ts`) cannot be
   deleted — `npm run typecheck` covers tests and reds instantly.
2. Live-feature tests (memory, compliance, review-status, conversations-favorite,
   projection, post-review-rebase) still bootstrap a **panopticon.db their production
   code never touches** — they pass while exercising a parallel dead schema, which is
   exactly the false-comfort the two-door tenet exists to kill.
3. Four UAT engine tests and the cost CLI test import types from dead modules whose
   type homes already exist in `src/lib/overdeck/`.

## 3. Locked design decisions

- **D1 — Order: buckets B → C → D → A+deletion.** Repoints and swaps land first, each
  commit green; the final commit deletes the 18 modules together with Bucket A's test
  files (they red typecheck if separated). One bucket (or coherent slice of C) per
  commit.
- **D2 — `index.ts` and `schema.ts` die with the stores.** After buckets B–D, their
  only importers are the cluster itself and Bucket A tests (verified: no production
  import of `database/index.js` or `database/schema.js` outside `src/lib/database/`).
  `getDatabasePath`/`getDatabase`/`closeDatabase`/`resetDatabase`/`initSchema`/
  `runMigrations` disappear — grep-verify zero references before the deletion commit.
- **D3 — Keep `driver.ts` + `conversation-embeddings-db.ts` + their two tests**;
  `src/lib/database/` survives as the generic-SQLite home (issue acceptance). Do NOT
  relocate them in this issue — moving the driver would churn 14 importers for zero
  behavior (Karpathy surgical rule).
- **D4 — Guard against regrowth.** In the deletion commit, update
  `scripts/lint-overdeck-boundaries.sh`: (a) its check #2 (importing old store
  modules) keeps working with the modules gone — verify it still greps by path
  pattern, not a file list; (b) add a contents allowlist check: `src/lib/database/`
  may contain only `driver.ts`, `conversation-embeddings-db.ts`, and
  `__tests__/{driver,conversation-embeddings-db}.test.ts` — any new file there fails
  with "new stores go in src/lib/overdeck/". Update the header comment ("deleted
  wholesale at cutover" → done).
- **D5 — Ratchet reconciliation in the deletion commit.** Drop the
  `1650 src/lib/database/schema.ts` entry from `scripts/file-size-baseline.txt`
  (verified present); remove any deleted file from `eslint-any-allowlist.json`
  (check: `grep "lib/database" eslint-any-allowlist.json`). If PAN-2227's check-mode
  is live it forces this; do it regardless.
- **D6 — No behavioral change to any live feature.** Buckets B/C/D touch only test
  files (plus the one test move). The only `src/` deletions are the cluster modules;
  the only `src/` non-deletion edit is the boundary-gate script (D4). If a swap
  reveals a live-feature test that FAILS against overdeck (not a scaffolding issue but
  a real behavior difference), STOP and surface it — that is a production bug found,
  not a test to massage (investigate-root-cause rule).

## 6. Intersecting repo rules (restated — do not assume recall)

- **Typecheck covers tests** — never delete a module before its last test binding is
  gone (D1 ordering is mandatory, not stylistic).
- **PAN-1988**: `*_notes` are journal-only — migrated review-status tests assert DB
  absence, not round-trip.
- **No-loss discipline**: deletions are audited (§2 tables); a test's guarantee is
  ported or its death is justified in the commit body — never silent.
- **Fix the system**: a live-feature test failing against overdeck.db is a production
  finding — stop and surface, don't adapt the test (D6).
- **Fake timers** for delay/retry tests (NFR-2). **No `execSync`** in anything
  server-reachable. **Keep the .db FILES** — this issue deletes code only.
- **commitlint**: all-lowercase, scope `db`; never `--no-verify`; never `git stash`;
  feature workspace, never the primary worktree.
