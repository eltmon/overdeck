## Problem

The Command Deck's right-pane shell (`ProjectRightPaneTabs`) shows one thing at
a time. It is a strip of six project tabs — Pipeline, Plans, Beads,
Conversations, Activity, Settings — and selecting a session swaps the whole
pane to a `ZoneB` + `ZoneCConversation` view. To compare a conversation against
the diff, or watch a terminal while reading the plan, you constantly swap the
single pane back and forth and lose your place.

Two problems compound:

1. **No persistence of place.** There is no notion of "the things I have open."
   Every tab/session switch replaces the view; scroll position, tab selection,
   and terminal focus are lost.
2. **The naming is spatial, not semantic.** `Zone A / Zone B / Zone C` describe
   *where a thing sits*, not *what it is*. (The most-confusing name,
   `ZoneCOverviewTabs`, turns out to belong to the dead `IssueWorkbench` path —
   so the live shell carries the same naming debt without the user ever seeing
   that component.)

## Vision

Replace the live single-pane `ProjectRightPaneTabs` shell with a **Stage** that
holds a persistent **PaneBar** of tabs. Each workspace opens on a permanent **HOME pane** (the
project overview + a smart command launcher) and the user opens additional
panes — agent conversations, terminals, commit/plan/docs views — that
persist independently and switch with `⌘1`–`⌘9`. This is the Subspace model,
adapted to Overdeck's data.

While we are in here, retire the spatial `Zone*` vocabulary for semantic names.

## Pane taxonomy & where the old tabs go

```typescript
type PaneType = 'home' | 'agent' | 'terminal' | 'commits' | 'plan' | 'docs'
              | 'files' | 'browser';   // 'files' + 'browser' implemented in PAN-1550
```

| paneType | Renders | Source component |
|---|---|---|
| `home` | HomePane — header, launcher, docks, timeline | **new**; stats compose from the read model; Timeline ≈ live Conversations tab |
| `agent` | AgentPane — conversation + terminal/findings toggle | live `ZoneB` + `ZoneCConversation` |
| `terminal` | TerminalPane — `XTerminal` on a tmux session | live `XTerminal` |
| `commits` | CommitsPane — git log + diff viewer | `PrDiffTab` (component exists; surfaced only in the dead path today) |
| `plan` | PlanPane — vBRIEF (List/DAG/Raw) + Beads | live `VBriefTab` (Plans tab) + `BeadsTab` (Beads tab) |
| `docs` | DocsPane — markdown viewer | `MarkdownTab` (PRD/STATE/INFERENCE; new surface) |
| `files` | _deferred_ | **→ [#1550](https://github.com/eltmon/overdeck/issues/1550)** |
| `browser` | _deferred_ | **→ [#1550](https://github.com/eltmon/overdeck/issues/1550)** |

`CostsTab` and `DiscussionsTab` are **not** pane types — they become collapsible
**HomePane sections** (Discussions preserves the live `DiscussionsTab` behavior;
Costs is additive, since `CostsTab` lives only in the dead path today). The tab
*bodies* are well-structured already, so the work is **re-homing** existing
components into panes/sections, not rebuilding them.

## Data model

```typescript
type PaneType = 'home' | 'agent' | 'terminal' | 'commits' | 'plan' | 'docs'
              | 'files' | 'browser';   // 'files' + 'browser' implemented in PAN-1550

interface WorkspacePane {
  paneId: string;                  // crypto.randomUUID()
  paneType: PaneType;
  label: string;
  createdAt: number;
  isPermanent?: boolean;           // true for home
  // agent pane
  conversationId?: string;
  agentType?: string;
  isRunning?: boolean;
  viewMode?: 'conversation' | 'terminal' | 'findings';
  // terminal pane
  terminalId?: string | null;
  // files / commits pane
  fileBrowserScopePath?: string;
  fileBrowserInitialQuery?: string;
  // docs pane
  docFilePath?: string;
  // browser pane
  browserInitialUrl?: string;
}

interface StageState {
  panes: WorkspacePane[];
  activePaneId: string;
}
```

State lives in a **new sibling Zustand store** at
`src/dashboard/frontend/src/lib/panesStore.ts` (following the
`lib/commandDeckSelection.ts` precedent — kept out of the event-sourced
`lib/store.ts`) and persists to localStorage under `pan-panes:{workspaceId}` and
`pan-active-pane:{workspaceId}`. The HOME pane is synthesized if absent and
cannot be closed.

## Command Launcher (the omnibox)

The most interactive part of HOME. Typing opens a dropdown of **quick actions**,
one row per intent, first row auto-selected:

```
┌──────────────────────────────── ↵ Run ─┐
│  ⌘  this is a test                      │
└─────────────────────────────────────────┘
│ 🟠  Ask Claude Code:        ⌘ ↵   │  ← auto-selected
│ ▢   Run in terminal:        ⌃ ↵   │
│ 🌐  Search the web:         ⌥ ↵   │
│ ⚙   Ask Codex:           ⌘ ⇧ ↵   │
```

**Intent ordering**
- Default (text): `claude-code` ⌘↵ → `terminal` ⌃↵ → `web` ⌥↵ → `codex` ⌘⇧↵
- URL-detected: `web` floats to position 1; the rest follow.
- **Last-used agent memory:** if the user last ran a non-Claude agent, that
  agent takes position 1 (so ⌘↵ stays "repeat last action").

**Label rules:** `terminal`→"Run in terminal:", `web`→"Search the web:", any
agent→"Ask {agentName}:".

**Behaviors:** dropdown hidden when input is empty; rows use
`onMouseDown + preventDefault` (clicking must not blur the input first); a
"compact/suppressed" mode renders only the quick-action rows (no history, no
file completions) for embedding elsewhere.

**Hard constraint — do not grow the monster.** `CommandDeck/index.tsx` is
already ~1603 lines with 30+ `useState`. Pane state goes in the new sibling
store and new `Stage/*` components; the only change to `index.tsx` is to swap
the `ProjectRightPaneTabs` render for `<Stage>` at the capstone. The HOME stats
(days, diff, files, convos) compose from data already in the read model
(workspace `createdAt`, git enrichment diff, changed-files list, conversation
count).

## Phased delivery — build-then-swap

This is one feature; the phases are implementation order, **not** separate
deliverables (per the repo's complete-features rule). **22 beads** ship before
`pan done` (the original 24 minus `files-pane` and `browser-pane`, deferred to
[#1550](https://github.com/eltmon/overdeck/issues/1550)). The delivery
strategy is **build-then-swap**: phases 1–5 add the Stage as **unmounted**
components, so every bead merges with a green tree and an **unchanged on-screen
UI**. Only the capstone swaps the mount and deletes the dead code.

1. **Foundation (unmounted)** — `panes-store`, `panebar`, `stage-shell`,
   `keyboard-shortcuts`.
2. **HomePane shell** — `homepane-shell`, `workspace-header`, `home-stats`.
3. **Launcher** — `launcher`, `launcher-ordering`, `launcher-wiring`.
4. **Pane wrappers** — `agent-pane`, `terminal-pane`, `commits-pane`,
   `plan-pane`, `docs-pane`. _(`files-pane`, `browser-pane` → #1550)_
5. **Docks, Timeline, sections** — `agent-dock`, `action-dock` (Terminal +
   Commits only — no Files/Web), `timeline`, `homepane-sections` (fold
   Overview/Activity/Discussions/Costs in).
6. **Capstone (the only visible change)** — `mount-stage` swaps
   `ProjectRightPaneTabs` → `<Stage>`; `delete-legacy` removes the audited dead
   shell (`IssueWorkbench` / `ZoneCOverview` / `ZoneCOverviewTabs`);
   `update-tests` replaces `CommandDeck.test.tsx` coverage.

## Decisions (resolved by `--auto` planning)

The planner recorded these in `plan.autoDecisions[]`; they settle the questions
earlier drafts left open:

1. **Target the live shell.** Replace `ProjectRightPaneTabs`, not the dead
   `IssueWorkbench`/`ZoneCOverview` path.
2. **Sibling store.** `lib/panesStore.ts`, not `src/store/` (no such dir).
3. **Project-vs-workspace boundary.** Stage is workspace/issue-scoped; the
   `pipeline` project kanban stays a project landing (`ProjectOverview`), **not**
   a pane. (resolves the old Q5)
4. **Build-then-swap** delivery (above), to keep every bead mergeable.
5. **"Stage" confirmed; Navigator (Zone B) rename deferred** as cosmetic churn.
6. **Costs & Discussions = HomePane sections** (reuse `CostsTab`/`DiscussionsTab`),
   not new pane types.
7. **First-load migration: HOME only** — do not auto-reopen the last
   conversation (deterministic/testable; Timeline gives one-click access).
8. **FilesPane / BrowserPane fully deferred** to
   [#1550](https://github.com/eltmon/overdeck/issues/1550) — **not** minimal
   stubs in this issue (supersedes the original auto-decision, which kept stubs).
   The ActionDock ships with **Terminal + Commits + ＋Actions** only; **Files**
   and **Web** buttons arrive with #1550. The `files-pane` and `browser-pane`
   beads are closed as deferred. The HOME stats row still *displays* `+X −Y` /
   `N files` (read-model data); its click-to-open-files action arrives with #1550.
