# PAN-1549 — Workspace Panes: a Stage/Pane shell with a HOME overview

> **Status:** Draft for discussion (2026-05-28). Scope confirmed as **full
> replacement** — the new pane system supersedes both `ProjectRightTab` and
> `ZoneCOverviewTabs` rather than living beside them.

## Problem

The Command Deck shows one thing at a time. Zone C toggles between a single
conversation (`ZoneCConversation`) and an issue overview (`ZoneCOverview`),
and the overview itself is a strip of **ten** sub-tabs under
`ZoneCOverviewTabs/` (Overview, Activity, Costs, PRD, STATE, INFERENCE,
vBRIEF, Beads, PR/Diff, Discussions). To compare a conversation against the
diff, or watch a terminal while reading the plan, you constantly swap the
single pane back and forth and lose your place.

Two problems compound:

1. **No persistence of place.** There is no notion of "the things I have open."
   Every navigation replaces the view; scroll position, sub-tab selection, and
   terminal focus are lost.
2. **The naming is spatial, not semantic.** `Zone A / Zone B / Zone C` and
   `ZoneCOverviewTabs` describe *where a thing sits*, not *what it is*. New
   contributors cannot guess what `ZoneCOverviewTabs` renders, and the names
   give no hint about the data each region shows.

## Vision

Replace the single-pane Zone C with a **Stage** that holds a persistent
**PaneBar** of tabs. Each workspace opens on a permanent **HOME pane** (the
project overview + a smart command launcher) and the user opens additional
panes — agent conversations, terminals, file/commit views, plan/docs — that
persist independently and switch with `⌘1`–`⌘9`. This is the Subspace model,
adapted to Panopticon's data.

While we are in here, retire the spatial `Zone*` vocabulary for semantic names.

## Naming model (top-down)

```
Command Deck            shell — keep the name
├── Navigator           left sidebar: projects ▸ workspaces ▸ convos   (was Zone B)
└── Stage               the main work area                            (was Zone C)
    ├── PaneBar         persistent tab strip                          (new)
    │   └── PaneTab     one tab, backed by a Pane
    └── Pane            active content, keyed by paneType
        ├── HomePane    workspace overview (⌘1, permanent)            (was ZoneCOverview)
        │   ├── WorkspaceHeader   name · branch · stat chips          (absorbs Zone A)
        │   ├── Launcher          smart omnibox                       (was "Command Launcher")
        │   ├── AgentDock         agent pill launchers
        │   ├── ActionDock        Terminal / Files / Web / Commits
        │   └── Timeline          conversation history
        ├── AgentPane   a conversation + terminal/findings toggle     (was ZoneCConversation)
        ├── TerminalPane · FilesPane · CommitsPane · BrowserPane
        ├── PlanPane    vBRIEF (List/DAG/Raw) + Beads                 (folds in those tabs)
        └── DocsPane    PRD / STATE / INFERENCE markdown              (folds in MarkdownTab)
```

### Rename table

| Today | New | Note |
|---|---|---|
| `Command Deck` (shell) | `Command Deck` | Unchanged — established. |
| `Zone B` | `Navigator` | Left sidebar tree. |
| `Zone A` (issue header) | folds into `WorkspaceHeader` | Id / title / actions move into the HOME header + PaneBar context. |
| `Zone C` (the area) | `Stage` | Hosts `PaneBar` + active `Pane` instead of a Conversation/Overview toggle. |
| `ZoneCConversation` | `AgentPane` | Reused as-is, rendered inside a pane. |
| `ZoneCOverview` | `HomePane` | The `⌘1` landing pane. |
| `ZoneCOverviewTabs/` (folder) | split into panes/sections | Tab bodies re-homed, not rewritten — see taxonomy. |

*("Workspace" was deliberately avoided for the Stage — it already means a git
worktree feature branch in Panopticon.)*

## Pane taxonomy & where the old tabs go

```typescript
type PaneType = 'home' | 'agent' | 'terminal' | 'files' | 'commits'
              | 'plan' | 'docs' | 'browser';
```

| paneType | Renders | Was |
|---|---|---|
| `home` | HomePane — header, launcher, docks, timeline | Overview tab + Activity feed |
| `agent` | AgentPane — conversation + terminal/findings toggle | `ZoneCConversation` |
| `terminal` | TerminalPane — `XTerminal` on a tmux session | — |
| `files` | FilesPane — file browser / recent docs | — |
| `commits` | CommitsPane — git log + diff viewer | PR / Diff tab |
| `plan` | PlanPane — vBRIEF (List/DAG/Raw) + Beads | vBRIEF tab + Beads tab |
| `docs` | DocsPane — markdown viewer | PRD / STATE / INFERENCE (MarkdownTab) |
| `browser` | BrowserPane — embedded web view | — |

The tab *bodies* are well-structured already (`OverviewTab`, `VBriefTab`,
`BeadsTab`, `PrDiffTab`, `MarkdownTab`, …). The work is **re-homing** them into
panes/sections, not rebuilding them.

## Data model

```typescript
type PaneType = 'home' | 'agent' | 'terminal' | 'files' | 'commits'
              | 'plan' | 'docs' | 'browser';

interface WorkspacePane {
  paneId: string;                  // crypto.randomUUID()
  paneType: PaneType;
  label: string;
  createdAt: number;
  isPermanent?: boolean;           // true for home
  // agent pane
  conversationId?: string;
  agentType?: string;
  isRunning?: boolean;
  viewMode?: 'conversation' | 'terminal' | 'findings';
  // terminal pane
  terminalId?: string | null;
  // files / commits pane
  fileBrowserScopePath?: string;
  fileBrowserInitialQuery?: string;
  // docs pane
  docFilePath?: string;
  // browser pane
  browserInitialUrl?: string;
}

interface StageState {
  panes: WorkspacePane[];
  activePaneId: string;
}
```

State lives in a new Zustand slice and persists to localStorage under
`pan-panes:{workspaceId}` and `pan-active-pane:{workspaceId}`. The HOME pane is
synthesized if absent and cannot be closed.

## Command Launcher (the omnibox)

The most interactive part of HOME. Typing opens a dropdown of **quick actions**,
one row per intent, first row auto-selected:

```
┌──────────────────────────────── ↵ Run ─┐
│  ⌘  this is a test                      │
└─────────────────────────────────────────┘
│ 🟠  Ask Claude Code:        ⌘ ↵   │  ← auto-selected
│ ▢   Run in terminal:        ⌃ ↵   │
│ 🌐  Search the web:         ⌥ ↵   │
│ ⚙   Ask Codex:           ⌘ ⇧ ↵   │
```

**Intent ordering**
- Default (text): `claude-code` ⌘↵ → `terminal` ⌃↵ → `web` ⌥↵ → `codex` ⌘⇧↵
- URL-detected: `web` floats to position 1; the rest follow.
- **Last-used agent memory:** if the user last ran a non-Claude agent, that
  agent takes position 1 (so ⌘↵ stays "repeat last action").

**Label rules:** `terminal`→"Run in terminal:", `web`→"Search the web:", any
agent→"Ask {agentName}:".

**Behaviors:** dropdown hidden when input is empty; rows use
`onMouseDown + preventDefault` (clicking must not blur the input first); a
"compact/suppressed" mode renders only the quick-action rows (no history, no
file completions) for embedding elsewhere.

## Keyboard shortcuts

| Shortcut | Action |
|---|---|
| ⌘1 | Focus HOME pane |
| ⌘2–⌘9 | Focus pane N |
| ⌘T | Open new pane |
| ⌘W | Close current pane (not HOME) |
| ⌘↵ | Run with Claude Code (from launcher) |
| ⌃↵ | Run in terminal |
| ⌥↵ | Search web |
| ⌘⇧↵ | Run with Codex |

## Where this lives / migration

| Component | Location |
|---|---|
| Pane state (Zustand) | `src/dashboard/frontend/src/store/panesSlice.ts` (new) |
| Tab strip | `…/components/Stage/PaneBar.tsx` (new) |
| Stage mount | `…/components/Stage/index.tsx` (new) |
| HOME pane | `…/components/Stage/HomePane/index.tsx` (new) |
| Launcher | `…/components/Stage/HomePane/Launcher.tsx` (new) |
| Pane renderers | reuse existing `SessionPanel`, `XTerminal`, `VBriefTab`, `BeadsTab`, `PrDiffTab`, `MarkdownTab` inside pane wrappers |
| Mount point | `components/CommandDeck/index.tsx` — replace the Zone C Conversation/Overview switch |

**Hard constraint — do not grow the monster.** `CommandDeck/index.tsx` is
already ~1603 lines with 30+ `useState`. Pane state goes in the new Zustand
slice and new `Stage/*` components; the only change to `index.tsx` is to mount
`<Stage>` where the Zone C switch is today. The HOME stats (days, diff, files,
convos) compose from data already in the read model (workspace `createdAt`,
git enrichment diff, changed-files list, conversation count).

## Phased delivery

This is one feature; the phases are implementation order, **not** separate
deliverables (per the repo's complete-features rule). All phases ship before
`pan done`.

1. **Pane model + PaneBar** — Zustand slice, static strip with HOME, add/close/
   switch, ⌘N shortcuts, localStorage persistence.
2. **HomePane header + stats** — name, branch, age, diff, file count, convo
   count, wired to the read model.
3. **Launcher** — smart omnibox with the intent dropdown + ordering + last-used
   memory.
4. **AgentDock + ActionDock** — pill launchers open `agent` panes; tool buttons
   open `terminal`/`files`/`commits`/`browser` panes.
5. **Timeline** — date-grouped conversation history; click opens/focuses an
   `agent` pane.
6. **Retire the old surface** — fold `ZoneCOverviewTabs` bodies into
   `PlanPane`/`DocsPane`/HomePane sections, delete `ProjectRightTab` +
   `ZoneCOverview` + `ZoneCConversation` switching, update `CommandDeck.test.tsx`.

## Open questions

These are deliberately unresolved — flagged rather than silently decided:

1. **"Stage"** as the name for the main area (was Zone C) — confirm, or prefer
   another (Canvas / Workbench)? "Workspace" is out (collides with worktrees).
2. **Navigator rename (Zone B)** — do it in this issue, or leave Zone B and only
   rename the Zone C cluster to reduce churn?
3. **Costs** — its own `costs` pane, a HomePane stat that expands, or a
   `+ Actions` item?
4. **Discussions** — its own pane, a HomePane section, or part of `agent`?
5. **`pipeline` (project kanban)** is *project*-scoped, not workspace-scoped.
   Does it become a pane, or stay a Navigator-driven project view? This is the
   one `ProjectRightTab` entry that doesn't map cleanly onto a workspace pane —
   it likely defines the **project-vs-workspace boundary** for the Stage.
6. **Migration of open state** — on first load after ship, do we synthesize a
   HOME pane only, or also re-open the user's last conversation as an `agent`
   pane so nothing feels lost?
