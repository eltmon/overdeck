## Glossary

- **Legacy / source DB** ‚Äî the pre-rebrand `~/.panopticon/panopticon.db`, resolved by
  `getLegacyHome()` (`src/lib/paths.ts:19`). Read-only for this feature. **Will not exist on
  the dev machine** ‚Äî tests must build a fixture from the legacy DDL.
- **New / target DB** ‚Äî `~/.overdeck/overdeck.db`, resolved by `getOverdeckDatabasePath()`
  (`src/lib/overdeck/paths.ts:8`). The live read/write DB.
- **Write door** ‚Äî the single sanctioned write surface for the conversations domain:
  `src/lib/overdeck/conversations.ts` (the sync-compat functions `createConversation`,
  `setFavorite`, ‚Ä¶ and the Effect `ConversationWriter`). Per the single-source-of-truth
  tenet, route handlers must NEVER write the conversations / favorites / conversation_files
  tables directly ‚Äî they call functions in this module.
- **`conversation_files`** ‚Äî the table that holds a conversation's Claude session UUID as
  `locator`. The new `conversations` table has **no** `claude_session_id` column.
- **Agent-prefixed conversation** ‚Äî a `name` starting `agent-` / `planning-` / `specialist-`
  (`isAgentConversationName`, `conversations.ts:797`). Excluded from import by default.

## The two gaps the issue under-specifies (DECISIONS MADE HERE)

### Gap A ‚Äî neither existing write door can carry the import

The issue's AC4/AC5 say "write ‚Ä¶ through the existing write door (`createConversation` / `ConversationWriter.create`)" **and** preserve the full column mapping **and** never rely on a destructive DELETE. Those cannot all be true of the existing functions:

- `createConversation` (`conversations.ts:1153`) **destructively** runs
  `DELETE FROM conversations WHERE name = ?` before insert, and hardcodes
  `status='active'`, `archived_at=NULL`, `fork_retry_count=0`, `spawn_error=NULL`. It inserts
  only `id,name,cwd,issue_id,harness,model,effort,title,title_source,created_at(=now),tmux_session,status,fork_status,delivery_method`.
  It **drops** `created_at` (stamps now), `archived_at`, `ended_at`, `last_attached_at`,
  `total_cost`, `total_tokens`, and both FK columns. Routing imports through it would
  silently overwrite a same-named conversation and lose cost/archival history ‚Äî violating AC5.
- `ConversationWriter.create` (`conversations.ts:415`) is non-destructive
  (`onConflictDoNothing`) but takes an even smaller field set
  (`{name,cwd,model,effort,harness,issueId,title}`) and **cannot pass `claudeSessionId`**,
  so it can't satisfy AC6.

**Decision (D2):** Add a purpose-built, **non-destructive** write-door function in the same
module so the single-source-of-truth tenet still holds (extending the writer, not adding a
parallel direct-SQL path in a route):

- `importLegacyConversation(mapped)` ‚Äî inserts a conversation with the **full** mapped column
  set (createdAt ms, archivedAt, status, endedAt, lastAttachedAt, totalCost, totalTokens,
  forkStatus, sessionFile, title, titleSource, deliveryMethod, tmuxSession), mints a fresh
  UUID, and registers the `conversation_files` locator when `claudeSessionId` is present
  (reusing the existing `INSERT OR IGNORE INTO conversation_files` shape). FK columns are
  left NULL here (set in pass 2). Returns `{ uuid, legacyRowid }`.
- `setImportedConversationLinks(uuid, { handoffTargetUuid, clearedToUuid })` ‚Äî the pass-2
  FK setter (`UPDATE conversations SET handoff_target_conv_id=?, cleared_to_conv_id=? WHERE id=?`).
- Reuse the existing `setFavorite('conversation', name)` for AC9 ‚Äî no new favorites writer.

The route layer and `legacy-import.ts` call these functions; they do **not** write the tables directly.

### Gap B ‚Äî the shared driver has no read-only open

AC8 wants the source DB opened **read-only via the shared driver**, but
`OpenDatabaseOptions` (`driver.ts:29`) is `{ allowExtension?: boolean }` ‚Äî there is no
`readOnly`. The whole options object is already passed to `new DatabaseSync(path, options)`
(`driver.ts:293`) and node:sqlite honors `readOnly`, so:

**Decision (D3):** add `readOnly?: boolean` to `OpenDatabaseOptions`. That single type
addition is sufficient for the Node path (the dashboard server is **Node-22-only**; the bun
branch at `driver.ts:288` is never exercised by this server-side feature). The importer
`existsSync(path)`-checks first and returns the friendly "not found" state (AC2) **before**
attempting the open ‚Äî a read-only open of a missing file throws.

### Gap C ‚Äî Q4 (default checkbox state) ‚Äî RESOLVED, not escalated

**Decision (D1):** the dialog defaults to **all conversations selected**, with Select-all /
Select-none controls. This matches the issue author's stated default assumption and the
"carry all conversations over" intent. It is a trivially reversible UI default; planning does
not block on it.

## Architecture: one deletable module

Per the issue's "build it to be easy to delete", all import-specific logic lives in **one new
module** `src/lib/overdeck/legacy-import.ts` plus the two small write-door additions in
`conversations.ts` and the one-line driver type change. The whole legacy-import path
(including `getLegacyHome()`) is slated for removal at cutover ‚Äî keeping it in one file makes
that a clean delete.

`legacy-import.ts` responsibilities:
- `previewLegacyConversations(path)` ‚Üí opens legacy DB read-only (own handle, **never** the
  singleton), selects conversations, excludes `isAgentConversationName`, joins/derives
  last-activity, flags which have a `type='conversation'` favorite, returns the preview list.
- `importLegacyConversations(path, names)` ‚Üí two-pass orchestration:
  1. **Pass 1:** for each selected name not already present (idempotency probe by name, then
     by `claude_session_id` via `getConversationByClaudeSessionId`), map legacy‚Üínew and call
     `importLegacyConversation`; build the `{ legacyRowid ‚Üí newUuid }` map. A row whose
     `created_at` is empty/unparseable **fails that row** (reason recorded) ‚Äî never a zero insert.
  2. **Pass 2:** for each imported conversation with a legacy `handoff_target_conv_id` /
     `cleared_to_conv_id`, look the target up in the rowid‚Üíuuid map and call
     `setImportedConversationLinks`; if the target wasn't imported, leave NULL and add a per-row
     warning. (Two passes avoid FK-constraint violations regardless of `PRAGMA foreign_keys`.)
  3. **Favorites:** for each imported conversation whose name has a legacy
     `type='conversation'` favorite, call `setFavorite`; count carried.
  - Returns `{ imported: string[], skipped: {name,reason}[], failed: {name,reason}[],
    warnings: {name,reason}[], favoritesCarried: number }`.

## Column mapping (authoritative ‚Äî supersedes any ambiguity in the issue table)

Legacy ISO `TEXT` timestamps ‚Üí new `integer` ms via `new Date(iso).getTime()`.
Legacy INTEGER rowid `id` ‚Üí discard; mint `randomUUID()`; keep rowid‚Üíuuid map for FK remap.

| Legacy | New | Mapping |
| --- | --- | --- |
| `id` (rowid) | `id` (uuid) | discard; mint UUID; record rowid‚Üíuuid |
| `name,cwd,issue_id,model,effort,harness,delivery_method,spawn_error,fork_status,fork_error,fork_request,fork_fallback_reason,session_file,title,title_source,tmux_session` | same | copy (null-coalesced) |
| `status` | `status` (NOT NULL) | copy; if null derive `archived_at ? 'ended' : 'active'`. (Imports are historical; the liveness oracle reconciles any 'active' import to 'ended' on its next poll ‚Äî no special-casing.) |
| `title_seed` | ‚Äî | **dropped** (no column in new schema) |
| `created_at` | `created_at` (NOT NULL, ms) | parse ISO‚Üíms; **empty/unparseable ‚áí fail the row** |
| `ended_at,last_attached_at,archived_at` | same (ms) | parse ISO‚Üíms (null passes through) |
| `total_cost,total_tokens,fork_retry_count` | same | copy (default 0 if null) |
| `claude_session_id` | `conversation_files.locator` | pass to `importLegacyConversation` as `claudeSessionId`; the function inserts the `conversation_files` row (`harness`, `locator`, `created_at`) |
| `handoff_target_conv_id` (rowid) | text UUID FK | **pass-2 remap** via rowid‚Üíuuid map; NULL + warning if target not imported |
| `cleared_to_conv_id` (rowid) | text UUID FK | same as above |
| `handoff_doc_path` | same | copy |
| legacy `favorites` (`type='conversation'`) | new `favorites` | `setFavorite('conversation', name)` for each imported conv whose name matches `item_id`; count reported |

## Backend endpoints (`src/dashboard/server/routes/settings.ts`)

Follow the existing `HttpRouter.add` pattern (`settings.ts:780`/`806`), register both in
`settingsRouteLayer` (`settings.ts:1078`).

- `GET /api/settings/legacy-import/conversations?path=<legacyDbPath>` ‚Äî `path` optional;
  default `join(getLegacyHome(),'panopticon.db')`. If the file is missing ‚Üí `200` with
  `{ found:false, defaultPath, message:"No legacy database found at <path>" }`. If present ‚Üí
  `{ found:true, path, conversations: PreviewRow[] }`. Read-only; must not touch the live
  `overdeck.db` connection.
- `POST /api/settings/legacy-import/conversations` ‚Äî body `{ path, names: string[] }`. **Must**
  call `rejectUnsafeDashboardMutationRequest(request)` first (CSRF). Returns
  `{ imported, skipped, failed, warnings, favoritesCarried }`.

`PreviewRow`: `{ name, title, createdAt, model, lastActivityAt|null, messageCount|null,
hasFavorite, claudeSessionId|null, alreadyImported }`.

## Requirements

**Functional**
- **FR-1** ‚Äî A button "Import conversations from old Panopticon" appears in Settings ‚Üí
  Experimental and opens the import dialog. (AC1)
- **FR-2** ‚Äî The preview endpoint lists legacy conversations read-only, excluding
  agent-prefixed names, showing name, title, created date, model, and last-activity/message
  count where derivable. (AC3)
- **FR-3** ‚Äî The import endpoint copies selected conversations into `overdeck.db` through the
  write door, preserving the full column mapping above (ISO‚Üíms timestamps, archived/ended/cost/
  tokens, FK remap, `claude_session_id` ‚Üí `conversation_files`). (AC4)
- **FR-4** ‚Äî Import is idempotent: a conversation already present (by name, or by
  `claude_session_id`/locator) is skipped-and-reported, never duplicated, never overwritten,
  and the importer never relies on `createConversation`'s destructive DELETE. (AC5)
- **FR-5** ‚Äî JSONL linkage is preserved and no backing transcript file is moved, copied, or
  mutated ‚Äî DB metadata only. (AC6)
- **FR-6** ‚Äî Legacy `type='conversation'` favorites for imported conversations are recreated
  via `setFavorite`; non-conversation favorites are ignored. (AC9)
- **FR-7** ‚Äî After import, the dialog reports imported / skipped / failed (per-row reasons) /
  favorites carried, and links to the Conversations panel. (AC7)
- **FR-8** ‚Äî Source-DB detection: default `~/.panopticon/panopticon.db`; a missing DB yields a
  clear "not found" state with a manual path picker (also the test/fixture hook). (AC2)

**Non-functional**
- **NFR-1** ‚Äî Async-only, no `execSync`; the source DB is read via the shared driver opened
  **read-only**, never by repointing the singleton. (AC8)
- **NFR-2** ‚Äî The POST import endpoint is guarded by the dashboard CSRF check
  (`rejectUnsafeDashboardMutationRequest` + `dashboardMutationJsonHeaders`).
- **NFR-3** ‚Äî Import-specific logic is isolated in `src/lib/overdeck/legacy-import.ts` for a
  clean delete at cutover.

## Restated repo rules that intersect this work

- **Single source of truth:** writes to conversations/favorites/conversation_files go ONLY
  through `src/lib/overdeck/conversations.ts`. The route and `legacy-import.ts` call its
  functions; they never write those tables directly. Reading the **legacy** DB directly is
  fine ‚Äî it is an external, non-canonical source (like git/GitHub), opened read-only.
- **Sacred JSONL:** never move/copy/delete/mutate `~/.claude/projects/**/*.jsonl`. Import
  copies DB metadata only.
- **No execSync in server code; async only.**
- **Dashboard server is Node-22-only** ‚Äî the bun branch in `driver.ts` is not exercised here.
- **No bandaids:** do not paper over the lossy write door ‚Äî add the proper non-destructive
  importer; do not repoint the singleton DB handle.
