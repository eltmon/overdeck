## Glossary

- **Session row** — a `SessionNode` entry under a feature in the Command Deck issue
  tree (`GET /api/session-trees`), rendered by `FeatureItem.tsx`.
- **Remote agent** — a work agent running on a Fly.io VM, with state at
  `~/.panopticon/agents/agent-<issue>/remote-state.json` (`location: 'remote'`).
- **Convergence gate** — review-cycle rule: findings on code unchanged since the
  prior approved synthesis SHA cannot be promoted to blockers; once the file
  changes again, they can.

## Requirements

- **FR-1 (blocking review finding):** the workspace working copies
  `.pan/continue.json` AND `.pan/spec.vbrief.json` must not be tracked on the
  branch. Both are gitignored (`.gitignore` lines 59–60 and 120/123, PAN-1073
  invariant) and untracked on `origin/main`, yet the branch tracks both — both
  currently carrying foreign PAN-1788 content. (The review's blocking finding
  names `continue.json` because only it appears in the PR diff;
  `spec.vbrief.json` is the identical violation inherited from the merge-base.)
  Required: remove both from the index via a new commit (no history rewrite) so
  `git ls-files -- .pan/continue.json .pan/spec.vbrief.json` returns nothing.
  Note: `git check-ignore` reports ignored only AFTER untracking — gitignore
  patterns do not apply to tracked files.
- **FR-2 (demoted correctness finding):** `fetchProjectSessionTree()`
  (`src/dashboard/server/routes/projects.ts:471–542`) seeds feature candidates
  only from local `workspaces/feature-*` directories. An active remote agent with
  `remote-state.json` but no local workspace directory produces no session row —
  recreating the original bug for remote-only agents. Required: also seed
  candidates from `listActiveRemoteAgentStates()`
  (`src/lib/remote/remote-agents.ts:82`), filtered to the requested project and
  deduplicated against the workspace scan.
