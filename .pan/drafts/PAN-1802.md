# PAN-1802 — `pan handoff` must degrade, not abort, on a large source conversation

**Issue:** [PAN-1802](https://github.com/eltmon/panopticon-cli/issues/1802)
**Mode:** `pan plan --auto` (defensible defaults recorded in vBRIEF `plan.autoDecisions[]`)

## Problem

`pan handoff <conv>` authors the successor conversation's seed by summarizing the
**source** conversation's full transcript. When the source is large, the summarizer
overflows the model context and the **entire handoff aborts**:

```
Handoff failed: Smart summary generation failed: … "result":"Prompt is too long" … "terminal_reason":"blocking_limit"
```

A conversation record is created (e.g. conv 2778) but its session never spawns,
leaving an orphan conversation marked `forkStatus='failed'` yet `status='active'`.
The most expensive, longest-running orchestrating conversation — exactly the one
that most needs to delegate — becomes unable to spawn *any* handoff.

   - `src/cli/commands/handoff.ts:88-105` calls the server; `:107-111` prints
     `Handoff failed: <forkError>` when the server reports `forkStatus='failed'`.

   - `:4284` `markConversationActive(newConv.name)`
   - `:4286-4291` `registerInFlightForkPipeline(runForkPipeline(…)).catch(() => updateForkStatus(name,'failed',msg))`
3. `runForkPipeline` (`conversations.ts:3862`), `forkMode==='handoff'`,
   `handoffAuthor==='external'`:
   - `:3938-3947` `authorHandoffExternal(...)` — can throw on overflow.
   - `:3948-3953` **catch → `buildSummary()` is called UNGUARDED.**
     `buildSummary` (`:3922-3928`) runs `generateSummaryForFork` →
     `adapter.compactSummary` → `generateSmartSummary`, which throws the SAME
     overflow. That throw escapes `runForkPipeline`.
   - `:3992` `ensureForkSessionReady(...)` (the spawn) is never reached.
4. Overflow origin:
   - `src/lib/conversations/smart-compaction.ts:1060-1066` wraps chunked
     summarization failures as `Smart summary generation failed: …`.
   - `generateSummaryFromPrompt` (`:791-821`) → `runModelSummary` →
     `runModelSummaryPromise` (`:693-789`). On `code !== 0` it rejects with
     `Summary generation failed: <stderr|stdout>` (`:755-758`); for an oversized
     prompt the `claude -p` JSON envelope (`"result":"Prompt is too long"`,
     `"terminal_reason":"blocking_limit"`) is the detail. **`is_error` is not
     inspected — overflow surfaces only as the non-zero exit.**
   - The chunker (`chunkEntriesByBudget` `:857-874`, `chunkSerializedTextByBudget`
     `:911-927`) splits on entry/turn boundaries but **sends a single
     entry/turn that exceeds the budget un-split** (its own comment at `:908-910`
     says so), and `generateSummaryFromPrompt` also prepends an accumulating
     `<previous-summary>` (`:805-809`) — either can overflow the window.

## Glossary

- **Source conversation** — the existing conversation being handed off *from*.
- **Successor** — the new conversation `pan handoff` creates.
- **Authored handoff doc** — Markdown the external authoring session writes
  (`authorHandoffExternal`, `summary-fork.ts:214`). Best-quality seed.
- **Summary fork** — fallback seed: an LLM structured summary of the transcript
  (`generateSummaryForFork`, `summary-fork.ts:522`).
- **Heuristic local summary** — `generateFallbackSummary` /
  `generateFallbackSummaryPromise` (`summary-fork.ts:445-520`). Reads the JSONL
  and extracts the first 10 user messages, files touched, and tools used — **no
  LLM call, so it cannot overflow a context window.**
- **Context overflow** — the `claude -p` `blocking_limit` / "Prompt is too long"
  condition (and the existing 400 "input exceeds the context window" tail).
- **`forkStatus`** — pipeline phase column on the conversation row
  (`'handoff' | 'summarizing' | 'spawning' | 'injecting' | 'failed' | null`).
  `null` = done/healthy.

## Proposal — a guaranteed-success degradation ladder

A `pan handoff` must **always** create and spawn a usable successor. Degrade
quality; never abort. The ladder:

1. **Authored handoff doc** (LLM, richest).
2. **LLM summary fork** — now *truncation-resilient*: when a summary chunk
   overflows, truncate it **head+tail** and retry once instead of aborting
   (fix #1). This also makes `authorHandoffExternal`'s precompaction resilient,
   so most large handoffs now succeed at tier 1/2.
3. **Heuristic local summary** — `generateFallbackSummary`, no LLM, cannot
   overflow.
4. **Focus-only seed** — empty summary + `prependFallbackFocus` so the
   operator's focus still carries (fix #2).

The fork pipeline always reaches `ensureForkSessionReady` and spawns; the
successor's `forkFallbackReason` is persisted so the CLI prints the yellow
fallback notice. If the pipeline *still* fails terminally (e.g. spawn itself
fails), the conversation is marked **ended** (not deleted — repo invariant) so it
does not linger as a live ghost (fix #3). Finally, `pan handoff --help` and the
`pan-handoff` skill document the 500-char focus limit and the auto-degrade
behavior (fix #4).

## Requirements

- **FR-1** `pan handoff` with an oversized source MUST NOT abort; it MUST always
  create AND spawn a usable successor conversation.
- **FR-2** When a summary chunk exceeds the summarizer model context, the
  summarizer MUST truncate the chunk (head+tail) and retry once rather than
  propagate the overflow.
- **FR-3** When every LLM summary attempt fails, the handoff MUST seed the
  successor from the heuristic local summary, and if that also fails, from the
  operator focus alone — then spawn.
- **FR-4** A fork pipeline that fails terminally MUST mark the conversation
  `ended` (not leave it `active`), so it is not a live ghost.
- **FR-5** `pan handoff --help` and `sync-sources/skills/pan-handoff/SKILL.md`
  MUST document the 500-char focus limit and the large-source auto-degrade
  behavior.
- **NFR-1** No behavior change for normal-sized handoffs; existing handoff /
  smart-compaction tests stay green.
- **NFR-2** The overflow detector is a pure, side-effect-free matcher that
  extends `src/lib/context-overflow.ts` (no new module).

## Work items (→ beads)

1. **overflow-detector** (FR-2, NFR-2) — extend `src/lib/context-overflow.ts`
   with `isContextOverflowError(err)` matching `prompt is too long` /
   `blocking_limit` (case-insensitive) plus the existing
   `CONTEXT_OVERFLOW_PATTERNS`. Pure matcher.
2. **summarizer-truncate-retry** (FR-2) — in `smart-compaction.ts`
   `generateSummaryFromPrompt`, on a context-overflow rejection, head+tail
   truncate the serialized chunk (and cap `previousSummary`) to a per-model char
   ceiling derived from `getChunkBudgetChars`, then retry once. Add a head+tail
   truncation helper. Depends on #1.
3. **handoff-always-spawns** (FR-1, FR-3) — in `runForkPipeline`
   (`conversations.ts`), guard every fallback `buildSummary()` so a throw drops
   to `generateFallbackSummary` (heuristic), and a further throw drops to an
   empty/focus-only seed; always continue to `ensureForkSessionReady`. Persist
   `forkFallbackReason`.
4. **mark-failed-fork-ended** (FR-4) — in the fork route's pipeline `.catch`
   (`conversations.ts:4288-4291`), also `markConversationEnded(newConv.name)`.
5. **document-limits** (FR-5) — update the handoff `.description()`/help in
   `src/cli/index.ts:433-441` and the `Fallback behavior` section of
   `sync-sources/skills/pan-handoff/SKILL.md`; keep `npm run lint`
   (`scripts/lint-skills.sh`) green.

## Non-goals

- The legacy in-process `createSummaryForkPromise` lib path is **out of scope** —
  it has no production callers (only a comment reference in
  `src/cli/commands/fork-client.ts:10`); the live path is `runForkPipeline`.
- No change to chunk *budgets* (`CHUNK_BUDGET_CHARS_BY_MODEL`) or to which model
  summarizes — only overflow *resilience*.
- No deletion of conversations (repo invariant: conversations are never deleted,
  only archived — `conversations.ts:2604-2606`).
- No change to the `--author source` (live-agent) authoring mode beyond the
  shared fallback guard it already inherits.
