# PAN-2153 — Decompose `routes/specialists.ts` (1753 lines) into focused modules

**Issue:** https://github.com/eltmon/overdeck/issues/2153
**Type:** Refactor — codebase-health. Behavior-preserving, pure relocation.

## Goal

Split `src/dashboard/server/routes/specialists.ts` (1753 lines) into cohesive
submodules in a sibling directory `src/dashboard/server/routes/specialists/`, each
under 1,000 lines. Keep `routes/specialists.ts` as a thin **barrel** that re-exports
the public symbols (so every existing importer resolves unchanged) and composes
`specialistsRouteLayer`. Pure moves + re-exports. No logic edits, no renamed exports,
no call-site changes.

## Requirements

- **FR-1** — New submodules live in sibling dir `src/dashboard/server/routes/specialists/`; each new file is under 1,000 lines.
- **FR-2** — All existing exports still resolve from `routes/specialists.ts` (barrel): `specialistsRouteLayer` (named + default), `_serverManagedMerges`, `firePostMergeLifecycle`, `validateSpecialistAutoCompleteMetadata`. Zero importer edits.
- **FR-3** — Behavior-preserving: pure moves + re-exports only; helper/route bodies unedited.
- **FR-4** — Route registration order is identical (each sub-layer keeps today's const order; barrel merges legacy then project; order-sensitive pairs intact).
- **FR-5** — The one source-text test (`tests/lib/cloister/review-agent.test.ts`) is repointed to the new module path in the same commit as the move.
- **NFR-1** — After each seam: `npm run typecheck` + `npm run lint` + the FULL suite (`npx vitest run --configLoader runner`) green (0 failed).
- **NFR-2** — A1 ratchet: no new explicit `any`; the file that receives the moved `any` (`legacy-routes.ts`) is added to `eslint-any-allowlist.json`. Async only (no `execSync`).

## Glossary

- **Barrel** — `routes/specialists.ts` after the refactor: imports the two route
  sub-layers from submodules, re-exports the public symbols, composes the route layer.
- **Sub-layer** — a `*RouteLayer = Layer.mergeAll(...)` exported by a submodule
  (`specialistsLegacyRouteLayer`, `specialistsProjectRouteLayer`). The house pattern in
  `routes/workspaces.ts`.
- **Shared internals** — types, helpers, the in-flight guard, and the two module-level
  singletons (`postMergeGuard`, `_serverManagedMerges`) plus the exported functions that
  close over them.
- **File-size guard** — `scripts/lint-file-size.sh` (run by `npm run lint`). Ceiling 1000
  for NEW files; baselined files (`scripts/file-size-baseline.txt`) may shrink but never
  grow. `specialists.ts` is baselined at 1753 — shrinking is allowed.
- **A1 ratchet** — `npm run lint` forbids new explicit `any`; files carrying `any` must be
  listed in `eslint-any-allowlist.json`.

## Decision rules

- **Singletons stay in one place.** `postMergeGuard` and `_serverManagedMerges` move to
  `shared.ts` once. `legacy-routes.ts` (the done handler) and external `merge-ops.ts`
  import the same module → single instance preserved. The barrel re-exports
  `_serverManagedMerges`/`firePostMergeLifecycle` from `shared.ts`.
- **Order is preserved.** Each sub-layer lists its route consts in today's relative order;
  the barrel merges legacy then project. Keep order-sensitive pairs intact:
  `/runs/:runId/stream` before `/runs/:runId`; `postProjectReviewRestartRoute` immediately
  before `postProjectReviewerRoleRestartRoute`.
- **A1:** add `routes/specialists/legacy-routes.ts` to `eslint-any-allowlist.json` (it
  receives the four moved `any`). Leave the existing `routes/specialists.ts` entry (inert).
- **No new `any`, async only (no `execSync`).** Pure relocation — none introduced.
- **Sequential seams.** All three beads mutate the same barrel and each depends on the
  prior, so they run shared → legacy → project, with the full gate after each.
