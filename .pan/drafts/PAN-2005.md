# PAN-2005 — Backlog Pickup Forecast UI

> Issue: [PAN-2005](https://github.com/eltmon/overdeck/issues/2005). Depends on PAN-2006's
> shared module. Mockups: `docs/design/mockups/backlog-pickup-forecast-fullui-v2.html`
> (canonical), `-explained.html`, `-fullui.html` (V1). Consumes `src/lib/backlog/pickup.ts`.

## Goal

Add a **Pickup Forecast** mode to the backlog screen: visualize the order the Flywheel
will pick up work (waves + lanes), a "Plan next" rail that suggests + fires planning, an
editor drawer to toggle pipeline state, and a concurrency control wired to `maxAgents`.
Never reimplement eligibility — import PAN-2006's module.

## Current state (verified)

- `BacklogSequencerPage.tsx` (`src/dashboard/frontend/src/pages/`) — fetches `GET /api/backlog/sequence`, filters to tiers, renders `BacklogDAG`. Has tier ribbon, filters, List/DAG toggle.
- `BacklogDAG.tsx` (`src/dashboard/frontend/src/components/backlog/`) — ReactFlow DAG, `IssueNode` (already restyled to mockup), `RationaleSidePanel` (detail drawer with importance/score/size/deps/flags/gate/planning/rationale + gate & planning SegControls). Node + edge styling already match the mockup (landed earlier).
- `/api/backlog/sequence` returns `rank,size,importance,score,condition,dependsOn,why,gate,planning,inPipeline,hasPrd,ready,title` (`backlog.ts:73-98`). No phase, no parked/vetoed/blocksMain yet (PAN-2006 adds them).

## Requirements (FR)

- **FR-1** Header **Dependency ⇄ Pickup Forecast** toggle; Forecast has **Waves** / **Lanes** sub-views.
- **FR-2** Waves: "Running now" (in-flight) → Wave 1..k (up to N eligible, rank order), pickup-sequence arrows. Cards reuse `IssueNode` styling + state chips (Ready/Planned/parked/vetoed/blocks-main).
- **FR-3** Lanes: N agent lanes, effort-width blocks, greedy schedule, makespan + long-pole highlight.
- **FR-4** "Plan next — highest leverage" rail: top needs-planning issues (blocks-main first), each with ⚡ Auto-plan / Plan… / ↗ Open.
- **FR-5** Editor drawer (extend `RationaleSidePanel`): toggles for Ready / pickup-gate (auto/promote/vetoed) / Park / Blocks-main; Planned derived badge + inline Auto-plan; action bar Auto-plan/Plan/Open. Writes via PAN-2006 WI-8 endpoints.
- **FR-6** State filter bar + counts (In flight / Ready / Planned / Blocks main / Parked / Vetoed / Needs planning).
- **FR-7** Concurrency control = Flywheel `maxAgents` (default); re-renders waves+lanes on change.
- **FR-8** Run chip in header (RUN-N · cohort x/y drained) from flywheel-run-state.
- **NFR** Light+dark via semantic tokens; eligibility from the shared module only.
