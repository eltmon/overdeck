# PAN-2146 — Decompose `src/lib/agents.ts` (4,572 lines) into focused modules

**Issue:** https://github.com/eltmon/overdeck/issues/2146
**Type:** Refactor — codebase-health (behavior-preserving)
**Backlog:** `docs/codebase-health/REMAINING-BACKLOG.md` §1 / §"Suggested priority" #1 (the biggest remaining offender)

## Glossary

- **Barrel** — A module whose only job is to `export ... from` other modules, so importers keep one stable import path while the implementation lives elsewhere. After this work, `src/lib/agents.ts` is a barrel.
- **Seam** — One cohesive cluster of exports (and their private helpers) extracted into a new sibling module in a single commit. One seam = one bead.
- **File-size guard** — `scripts/lint-file-size.sh` (wired into `npm run lint`). Ceiling = 1,000 lines for any non-baselined file. `agents.ts` is baselined at 5,824 in `scripts/file-size-baseline.txt` (shrink-only); it keeps passing as it shrinks. **New modules are NOT baselined and must stay under 1,000 lines on their own.**
- **A1 ratchet** — The "no new explicit `any`" lint rule. `eslint-any-allowlist.json` lists files permitted to retain pre-existing `any`. When code carrying an `any` moves to a new file, that new file must be added to the allowlist (the `any` is relocated, not introduced).
- **Full suite** — `npx vitest run --configLoader runner` (0 failed). This is the project's root test command (`package.json` → `"test"` runs the same plus three frontend specs).

## Problem

`src/lib/agents.ts` is **4,572 lines** with **73 exported symbols** and ~65 internal helpers / module-level state — the largest remaining backend god file and a hot path (spawn, resume, recover, message). The first codebase-health pass pulled only 5 small modules out (`src/lib/agents/{activity,delivery,queries,runtime-state,termination}.ts`); the bulk remains. The established extraction pattern is already in the repo (see the four existing re-export lines at `agents.ts:3343-3346`), so this is a *finish*, not a fresh start.

## Proposal

Extract the remaining exported clusters into new sibling modules under `src/lib/agents/`, each **under 1,000 lines**, and reduce `src/lib/agents.ts` to a **thin re-export barrel**. **Pure relocation:** move declarations verbatim, add a re-export line in the barrel, fix imports. **No logic edits, no renamed exports, no signature changes.** Every existing import of `src/lib/agents.ts` resolves unchanged because the barrel re-exports everything.

Run the full quality gate (`typecheck` + `lint` + full suite) **after each seam** so any breakage surfaces against the smallest possible diff. This is the safety mechanism that makes a behavior-preserving relocation verifiable; it replaces the need for an inspection gate.

### Decisions pinned for the executor

- **`Role` type + `isRole` move into `agents/agent-state.ts`** (alongside `AgentState`), making that module the type foundation. The barrel re-exports both, so existing submodules that import `Role`/`AgentState` from `'../agents.js'` (e.g. `src/lib/agents/queries.ts:6`) keep resolving unchanged. New submodules import these from `./agent-state.js` directly.
- **Extraction order is leaf-and-foundation-first** so no new submodule ever needs to import from the barrel: `agent-state.ts` (foundation) is extracted **before** any module that consumes `Role`/`AgentState` (runtime-command, provider-env, supervisor-channels, spawn, messaging, resume, recovery). `identity.ts` is pure and may go anytime.
- **The spawn and recovery seams are split unconditionally** — measured content is ~1,320 lines (spawn) and ~919 lines (recovery), both over the 1,000 ceiling once a module import header is added. Spawn splits at the `spawnRun` boundary into `spawn-prep.ts` + `spawn.ts`; recovery splits at the `restartAgent` boundary into `resume.ts` + `recovery.ts`.
- **No-loss audit gate first.** Before any code moves, add a frozen export-surface snapshot test (`src/lib/__tests__/agents-barrel-exports.test.ts`) that asserts the exact sorted set of named exports of `src/lib/agents.ts` against a captured literal list. Every later seam must keep it green — this is the mechanical proof that the public surface is preserved (satisfies the additive-refactor / no-loss rule).

Private helpers and module-level constants/state move with their **sole** consumer; helpers used by **two or more** new modules go into a new `src/lib/agents/shared.ts` (kept < 1,000 lines) and are imported from there.

## Requirements

- **FR-1** Each new submodule under `src/lib/agents/` is < 1,000 lines (file-size guard ceiling).
- **FR-2** `src/lib/agents.ts` becomes a thin re-export barrel; every symbol previously exported from it still resolves from it unchanged (zero importer edits across `src/`, `tests/`, and the frontend).
- **FR-3** Behavior-preserving: pure moves + re-exports only. No logic edits, no renamed exports, no signature changes.
- **FR-4** After each seam: `npm run typecheck` + `npm run lint` + the full suite (`npx vitest run --configLoader runner`, 0 failed). Any test that deep-imports a moved internal is repointed in the same commit.
- **FR-5** No-loss audit gate: `src/lib/__tests__/agents-barrel-exports.test.ts` freezes the barrel's named-export set and stays green across every seam.
- **NFR-1** A1 ratchet: no new explicit `any`. The six pre-existing `any` move with their code into `spawn.ts` (3) and `resume.ts`/`recovery.ts` (3); add those new files to `eslint-any-allowlist.json` in the seam that moves them.
- **NFR-2** Async only — do not introduce `execSync` (the existing `execSync` import moves verbatim with its current callers; no new sync calls). New submodules import sibling submodules **directly** (e.g. `from './agent-state.js'`), **never** from the barrel `../agents.js`, to avoid circular ESM that Node strict-ESM rejects at runtime.

## Acceptance

- `src/lib/agents.ts` is materially smaller (thin barrel).
- Every new submodule is < 1,000 lines.
- `npm run typecheck`, `npm run lint`, and the full suite are green.
- Behavior identical: the diff is pure relocation + re-export + import-path fixes (no edited logic).

## Hazards

- **Shared private helpers** (e.g. `markAgentRunning`/`markAgentStopped`, `transitionIssueState`, `FsError`/`toAgentFsError`, `execAsync`, `defaultRunWorkspace`, `runAgentId`) are used by more than one destination module. Route these through `src/lib/agents/shared.ts` rather than duplicating or leaving a cross-import to the barrel.
- **Circular imports** — submodules must import siblings directly, never via the barrel; a barrel re-import creates a cycle Node rejects. Run a quick boot/import smoke (the full suite exercises most import graphs) after the seams.
- **Module-level mutable state** (`spawnStackRebuildState`, `missingRoleDefinitionWarnings`, the ready-signal sets) must move with its sole owner so it is not silently re-instantiated in two places.
- **`__testInternals`** stays re-exported from the barrel so existing test imports keep resolving.
