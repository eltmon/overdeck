## Glossary (the locked vocabulary)

- **Ready** — operator says "this may enter the pipeline" (Definition of Ready). Mechanism: GitHub `ready` label / Linear `Todo` state. The hard *entry* gate.
- **Planned** — worked-out enough to start: has a vBRIEF spec **AND** beads. Derived, read-only. Distinct from Ready.
- **parked** — deferred, needs human design/discussion. Reversible. Replaces `needs-design` + `needs-discussion`. Excluded from auto-pickup.
- **vetoed** — absolute NO. Never pick up, plan, or strike — overrides even a Flywheel pipeline-unblock. The one hard stop. (Final name, operator-confirmed.)
- **blocks-main** — must land to green main. Flywheel prioritizes/strikes even with auto-pickup off (never if vetoed).
- **dependsOn** — prerequisite ordering (the *other* "blocked" sense). Affects rank, not a veto.
- **pickup gate** — operator pickup override stored in `.pan/backlog/sequence.md` `gate`: `auto` / `promote` / `vetoed`.

## Work items (ordered — single issue)

### WI-1 Foundation: shared eligibility module + types
- New `src/lib/backlog/pickup.ts` (pure, no I/O beyond injected lookups). Exports:
  - `type PipelineState = { ready: boolean; planned: boolean; parked: boolean; vetoed: boolean; blocksMain: boolean; inPipeline: boolean; gate: 'auto'|'promote'|'vetoed' }`
  - `classifyIssue(node, lookups): PipelineState` — derives the booleans from labels + spec/beads + sequence gate.
  - `isAutoPickable(state): boolean` = `ready && planned && !parked && !vetoed && !inPipeline && gate!=='vetoed'`.
  - `isUnblockEligible(state): boolean` = `blocksMain && !vetoed` (override path; bypasses `ready`/`auto_pickup`).

- Refactor `pickFromSequence` to call `classifyIssue`/`isAutoPickable` (no behavior reimplementation).
- Tests: `src/lib/backlog/__tests__/pickup.test.ts` — classification truth table, `isAutoPickable`/`isUnblockEligible`, vetoed-overrides-everything, wave batching, lane makespan, cohort selection. **Parity test**: `pickFromSequence` result == module's top auto-pickable for a fixture.
