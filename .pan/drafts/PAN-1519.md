# PAN-1519: Remove graphify integration entirely

**Tracking issue:** [#1519](https://github.com/eltmon/panopticon-cli/issues/1519)

## Summary

Remove every panopticon-owned trace of graphify: the merge-agent post-merge refresh hook, the `pan doctor` freshness check, the `## graphify` CLAUDE.md section, the `.claude/settings.json` PreToolUse hook that nudges agents toward the graph, the `src/lib/graphify/` module + tests, the on-disk `graphify-out/` directories across the main repo and all worktrees, the graphify references in the token-spend-report and pan-swarm skills, and the two open tracking issues (PAN-1039, PAN-1408). Drop the Graphify half of PAN-1481 and retitle it to Caveman-only.

The third-party `/graphify` skill at `/home/eltmon/.claude/skills/graphify/SKILL.md` stays — it's not panopticon-owned, and ad-hoc one-off graph queries remain available.

## Motivation

graphify was integrated into panopticon on 2026-05-08 (PAN-1039) on the bet that a knowledge graph would save agent tokens on cross-file exploration. After 16 days in production the bet has not paid off and the wiring has been actively harmful.

**The savings claim has never been measured.** `panopticon-cli/graphify-out/cost.json` records exactly one run (2026-05-09) with `total_input_tokens: 0, total_output_tokens: 0`. PAN-1481 — the issue that would actually wire cost-event telemetry — is open and unstarted.

**The graph has been stale for 16 days.** `GRAPH_SUMMARY.md` mtime is 2026-05-09; HEAD on `origin/main` has advanced 676 merges since.

**The merge-agent refresh is a 100% no-op.** `graphify-out/` is gitignored at `.gitignore:113`. `refreshGraphify()` runs `graphify update .` locally, then `git add graphify-out/` returns ``skipped: 'gitignored'`` and the helper silently returns. PAN-1408's "commit refresh on follow-up" acceptance criterion has never fired on origin/main.

**Agents have been reading stale data and paying tokens for it.** `CLAUDE.md:557-567` instructs every agent: "At task start, read `graphify-out/GRAPH_SUMMARY.md` (~700 tokens) for orientation." Across many sessions per day, that's a real recurring cost paying for May-9 worldview information.

**The integration plan barely shipped.** PAN-1039's design covered six layers — MCP server, watcher daemon, dashboard view, wiki export, cost telemetry, review/specialist context injection. What shipped: a skill, a broken merge-agent hook, a doctor check that warns about staleness nobody can fix.

**Disk footprint is concrete:** 136MB in `panopticon-cli/graphify-out/`, ~99MB across 38 worktrees, plus a 4.5GB orphan at `~/Projects/graphify-out/` (deleted earlier today). Total reclaim ~234MB.

## Design goals

1. Remove every code path, config entry, doc reference, and skill mention.
2. Keep the third-party `/graphify` skill installed so manual ad-hoc usage is still possible.
3. Preserve historical artifacts unchanged: `.release/v*.md` and `docs/audit-*` are frozen records of what shipped at the time.
4. Land in one PR so the build stays green at every commit on main and the rollback is a single revert.
