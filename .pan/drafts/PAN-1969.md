# PRD: Home shows registered projects + "New Project" entry point (PAN-1969)

> Part of the **minimum new-user onboarding** (demo readiness). Frontend-only. Opens the New Project flow built in **PAN-1970**.

## Summary
Make the Home "System briefing" page **list the registered projects** (it currently shows none) and add an obvious **"+ New Project"** entry point on Home and on the sidebar PROJECTS header.

## Glossary
- **Home / System briefing** — landing page `HomePage.tsx` (the `home` tab).
- **Registered project** — an entry in `~/.overdeck/projects.yaml`, surfaced via `GET /api/registered-projects`.
- **Sidebar PROJECTS** — the left-rail project list (`Sidebar.tsx`).

## Problem
`HomePage.tsx` only renders memory-derived "workspace" cards (`buildHomeWorkspaceCards`, 362-398) from `statusByIssueId` / `observationsByIssueId` / `agentsById`. It **never queries the project registry**, so with projects configured the Workspaces section shows *"No workspace status is available yet"* (165-169) and there's no way to create a project. The sidebar **does** list projects (`fetchProjects`, queryKey `['command-deck-projects']`) — Home just doesn't reuse it.

## Goal / Non-goals
- **Goal:** Home shows the registered projects (matching the sidebar), each click-through to its deck; a clear "+ New Project" affordance on Home and the sidebar PROJECTS header opens the New Project flow.
- **Non-goals:** building the New Project flow itself (PAN-1970); changing the existing memory "Workspaces" cards.

## Requirements
- **FR-1** — Home renders a **"Projects" section** listing all registered projects, reusing the `['command-deck-projects']` query so it matches the sidebar; each row click-through selects that project's deck.
- **FR-2** — A **"+ New Project"** button on the Home Projects section header.
- **FR-3** — A **"+"** button on the **sidebar PROJECTS** header (convert the `<p>` at `Sidebar.tsx:387` into a flex row + `<Plus size={13} />`, mirroring the New-conversation button).
- **FR-4** — Both "+ New Project" affordances invoke a single `onNewProject()` handler that opens the **PAN-1970** flow. Until PAN-1970 lands, wire to a stub that no-ops with a TODO (do not block this issue on PAN-1970).
- **FR-5** — **Empty state:** zero registered projects → Home shows a friendly "No projects yet — create your first one" prompt with the same "+ New Project" CTA (the first-run case).
- **NFR-1** — Reuse the existing `fetchProjects` / `['command-deck-projects']` query; no new endpoint.
