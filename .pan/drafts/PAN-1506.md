# PAN-1506 — fix(dashboard): strike agents missing from frontend store

**Status after discovery (2026-06-29): ALREADY FIXED AND SHIPPED — recommend verify-and-close, no implementation needed.**

## Summary

The issue (filed 2026-05-25) reported that strike agents (`strike-pan-*`) appeared in
`/api/agents` but never reached the dashboard frontend Zustand store. Discovery for this
planning session found the root cause was already fixed **the same day the issue was
filed**, and the fix has shipped to `main`. The issue is OPEN only because it was never
closed.

## Root cause (already fixed)

Strike agents were dropped by **role guards** that did not list `'strike'`, so a strike's
`role` was treated as invalid and stripped before the agent reached the read model /
frontend snapshot. The fix added `'strike'` to every guard layer:

- `isRole()` in `src/lib/agents.ts` — `parseAgentState()` returns `null` for unknown roles.
- `Role` literal in `packages/contracts/src/types.ts`.
- `VALID_ROLES` / `toRole()` in `src/dashboard/server/read-model.ts:226`.
- overdeck.db `Role` schema in `src/lib/overdeck/agents.ts:60` (hardened later by PAN-1979,
  whose comment notes a too-narrow Role enum had crashed the AgentsResolver list decode on
  real `strike`/`flywheel` rows).

Fix commit: `61814fe26` — *"fix(read-model): include 'strike' in role guards so strike
agents reach the dashboard"* (2026-05-25). Confirmed an ancestor of HEAD (merged to main).

## Why the issue body is stale

The issue's "Hypothesis" section blames a **projection-cache** snapshot path. That
architecture no longer exists. Since the fix, the read model was rewritten twice:

- **PAN-1908** — three-plane agent state model.
- **PAN-1938** — source-swap: the snapshot now sources agents from the **overdeck.db
  agents table** via `AgentsResolver.list({})`, not a projection cache.

Under the current architecture the strike survives end-to-end:
`spawnAgent` (`src/lib/agents.ts:2951`) → `saveAgentStateSync` (`:3040`) writes the
overdeck.db agents row with `role='strike'` **and** state.json → bootstrap
`AgentsResolver.list({})` → `agentSnapshotFromOverdeck` (`read-model.ts:394`, passes
`role` through) → `pruneAgentsForReadSource` keeps it → `buildSnapshot` includes it in
the `agents` array delivered over the WebSocket.
