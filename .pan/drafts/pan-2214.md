# PAN-2214 — Swarm per-issue freeze/stop/reset controls + reconcile-safe bounded slot allocator

**Issue:** https://github.com/eltmon/overdeck/issues/2214 · Planned 2026-07-02 (auto mode)

## Glossary

- **Swarm** — Deacon-coordinated parallel execution of one issue's vBRIEF items across numbered **slots** (`agent-<issue>-slot-N` tmux sessions, `feature/<issue>-slot-N` branches, `<workspace>-slot-N` worktrees).
- **Coordinator** — `coordinateSwarmSlots()` in `src/lib/cloister/deacon-swarm.ts:168`: reconcile → classify → merge → gc → dispatch, run per Deacon patrol and reactively on `agent.stopped` (`src/lib/cloister/service.ts:436`).
- **Slot assignments** — durable registry at `record.swarm.slotAssignments` in the per-issue record (`.pan/records/<issue>.json`, written via `writeIssueRecordForWorkspaceSync`).
- **Per-issue hold** — `stuck` / `deaconIgnored` flags on review status (`src/lib/review-status.ts:86,111`); honored by the coordinator since `cd7553b8e8`. Write door: `setDeaconIgnored()` (`src/lib/review-status.ts:984`).
- **Global freeze** — `deacon.globally_paused` app-setting (`isDeaconGloballyPausedSync()`, `src/lib/overdeck/control-settings.ts:462`), checked once at patrol-cycle start (`src/lib/cloister/deacon.ts:2573`).
- **Swarm reserve** — `concurrency.reserved_swarm_slots` (PAN-2212), the dedicated dispatch budget gated by `tryReserveSwarmSlot()` (`src/lib/cloister/concurrency.ts:198`).

## Verified root causes (2026-07-02, HEAD 952c48b034)

1. **No per-issue controls.** `pan pause <id>` resolves a single agent id (`src/cli/commands/pause.ts:12`) — a swarm has none, so it fails. The hold flags exist and the coordinator honors them (deacon-swarm.ts:178-182), but the only setter is a dashboard route (`routes/workspaces/review-control.ts:474`); no CLI, no stop/reset routine.
2. **Unbounded index climb.** `dispatchNextWave` builds `occupiedSlotIndexes` from reconciled inFlight + unmerged branches + agents only (deacon-swarm.ts:598-602). Orphaned worktrees on disk are invisible (PAN-2213); `duplicateSpawnReason` then *refuses* rather than trying the next index; `lowestFreeSlotIndex` (deacon-swarm.ts:785) has no upper bound — indexes climbed 5→13→20 in the PAN-1791 runaway.
3. **Cap fires at zero live slots.** `registeredSlotCapacityAvailable` (deacon-swarm.ts:802) counts `listAgentStates({role:'work'})` rows with status `starting|running` — the DB registration plane, not the tmux liveness oracle. Stale rows survive a hand-reset and block all dispatch ("registered slot cap reached" at zero live sessions). It also caps by `maxWorkAgents` while dispatch draws from the PAN-2212 swarm reserve — two inconsistent budgets.
4. **GC kills live slots.** `gcMergedSlots` (deacon-swarm.ts:222) treats `git branch --merged HEAD` output as merged. A freshly-spawned slot branch with zero commits ahead is merged-by-ancestry, so gc ran `worktree remove --force` + `branch -D` under a live slot-17 agent.
5. **Freeze not honored mid-cycle.** Global pause is checked once at cycle start; slot-20 dispatched mid-patrol during a freeze.
6. **Manual dispatch races and skips reconcile.** `swarmCommand` (`src/cli/commands/swarm.ts:78`) calls `dispatchNextWave` with an **empty** `SlotReconcileResult` — no reconcile, no hold/freeze check.

## Requirements

- **FR-1** `pan swarm freeze <id>` sets the per-issue hold (via `setDeaconIgnored` write door, reason recorded); `pan swarm resume <id>` clears it. Idempotent.
- **FR-2** `pan swarm stop <id>` = freeze + stop every live slot agent for the issue. Idempotent when nothing is running.
- **FR-3** `pan swarm reset <id>` = stop + clean slot state (worktrees, local slot branches, slotAssignments, failed-merge block, stale agent registrations) **without losing work**: unmerged slot branches are pushed to origin before local deletion; refuses (unless `--force`) when a push fails. Hold stays set afterward; dispatch works again after `pan swarm resume`.
- **FR-4** `pan swarm status <id>` prints the reconciled view: slot assignments, branches (merged/unmerged), live sessions, holds, cap/budget occupancy.
- **FR-5** `pan pause <id>` on an issue whose agents are swarm slots exits with actionable guidance naming `pan swarm stop <id>` (instead of "Agent agent-<id> not found").
- **FR-6** Slot allocation is **bounded and reconcile-safe**: occupied indexes derive from durable state including orphaned slot worktrees/branches on disk; on a duplicate-refusal the allocator advances to the next free index; no index above the swarm-reserve-derived bound is ever allocated — when none is free the item defers with a reason naming the blocking slots.
- **FR-7** The registered-slot cap counts **tmux-alive** slot sessions and caps by the swarm reserve; stale DB rows with dead tmux never block dispatch.
- **FR-8** GC never removes a slot whose tmux session is alive; zero-commit "merged-by-ancestry" branches of live slots are never gc'd.
- **FR-9** Truly-orphaned slots (no assignment, no live agent, no unmerged commits) are gc'd by the coordinator; orphans **with** unmerged commits are reported, never deleted.
- **FR-10** Global freeze and per-issue hold are re-checked immediately before every slot spawn (patrol and reactive paths, incl. `recoverFailedMergeSlot`), so an in-flight cycle stops dispatching once frozen.
- **FR-11** `pan swarm <id>` dispatches through the same door as the Deacon (`coordinateSwarmSlots({issueId})` after real reconcile) — idempotent with concurrent Deacon coordination.
- **NFR-1** All new commands are idempotent and re-runnable.
- **NFR-2** No `execSync` in server-reachable code; async primitives only.
- **NFR-3** Cooldown/delay tests use fake timers (`vi.useFakeTimers()`).
- **NFR-4** `sync-sources/skills/pan-swarm/SKILL.md` updated in the same commits as CLI changes (`scripts/lint-skills.sh` enforces).
