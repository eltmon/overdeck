### Per-AC verification
1. **Selecting Managed-archived in the Sessions page shows all `conversations.archived_at NOT NULL` rows**
   ✗ NOT MET  
   *Evidence*: Live DB has 709 archived rows:
   ```json
   {
     "count": 709,
     "rows": [
       { "name": "20260524-2902", "archived_at": "2026-05-24T02:13:33.711Z" }
     ]
   }
   ```
   But live endpoint/UI only returns/renders 50 by default:
   ```json
   {
     "archivedResponses": [
       {
         "url": "https://pan.localhost/api/conversations/archived?limit=50",
         "status": 200,
         "count": 50,
         "first": "20260524-2902"
       }
     ],
     "rowCount": 50
   }
   ```
   `/api/conversations/archived?limit=1000` returns only 100 because the route caps `limit` at 100 (`src/dashboard/server/routes/conversations.ts:1296`). The Sessions UI hardcodes `limit=50` for archived rows (`src/dashboard/frontend/src/components/conversations/ConversationsPage.tsx:381-396`) and has no pagination for managed-archived rows.  
   Screenshot: `/tmp/audit-pass3/screenshots/PAN-1391-managed-archived.png`

   ✗ NOT MET
