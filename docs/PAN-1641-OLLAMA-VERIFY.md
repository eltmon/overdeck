# PAN-1641 local Ollama Pi verification

Date: 2026-06-08

## Scope

This note records the end-to-end local-model verification for PAN-1641's Pi harness Ollama path. The verification used the bundled Panopticon Pi extension with Pi's `ollama/<tag>` model selector and Ollama's OpenAI-compatible `/v1` endpoint.

## Environment

- System Ollama binary: `/usr/local/bin/ollama` reports server `0.19.0`; this could not pull `gemma4:12b` because the host service is too old.
- Passwordless `sudo` is not available in this workspace session, so the system service was not modified.
- For verification, a current Ollama Linux amd64 binary was downloaded to `/tmp/pan1641-ollama-new/bin/ollama` and served on `127.0.0.1:11435`.
- Model pulled on the current local server: `gemma4:12b`.

## Commands run

```bash
OLLAMA_HOST=127.0.0.1:11435 /tmp/pan1641-ollama-new/bin/ollama serve
OLLAMA_HOST=127.0.0.1:11435 /tmp/pan1641-ollama-new/bin/ollama pull gemma4:12b
OLLAMA_HOST=127.0.0.1:11435 /tmp/pan1641-ollama-new/bin/ollama list
```

The pulled model was present:

```text
NAME          ID              SIZE      MODIFIED
gemma4:12b    4eb23ef187e2    7.6 GB    4 seconds ago
```

The Pi run used the compiled Panopticon extension and the same provider env shape generated by the launcher for Ollama models:

```bash
PANOPTICON_AGENT_ID=agent-pan1641-gemma-e2e \
OPENAI_BASE_URL=http://127.0.0.1:11435/v1 \
OPENAI_API_KEY=ollama \
PANOPTICON_OLLAMA_MODEL=gemma4:12b \
PI_OFFLINE=1 \
strace -f -e trace=connect -s 200 -o /tmp/pan1641-pi-gemma.strace \
  pi --offline --no-session --no-tools \
  --extension packages/pi-extension/dist/index.js \
  --model ollama/gemma4:12b \
  -p 'Respond with exactly PAN1641_GEMMA_LOCAL_OK and no other text.' \
  > /tmp/pan1641-pi-gemma.out
```

## Result

Pi completed the local Gemma request successfully:

```text
PAN1641_GEMMA_LOCAL_OK
```

`strace` observed only localhost connect targets:

```text
connect_count 3
127.0.0.1:3011
127.0.0.1:11435
non_local_count 0
```

The `127.0.0.1:3011` connections are the Panopticon extension heartbeat attempts to the local dashboard. The model request went to `127.0.0.1:11435`, the local Ollama OpenAI-compatible endpoint. No Anthropic, OpenAI, or other cloud endpoint connections were observed during the Pi model run.

## Conclusion

The Pi + Ollama path runs end-to-end against local `gemma4:12b` with zero cloud model calls. The remaining host note is operational: `/usr/local/bin/ollama` should still be upgraded outside this passwordless workspace session so the default `http://localhost:11434` system service can also pull and serve `gemma4:12b`.
