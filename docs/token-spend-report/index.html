Methodology

How we calculated these numbers

Data Sources

  • panopticon.db cost_events table — 734,117 tracked API calls across 464 days of active cost tracking (Feb 2025 – May 2026)
  • Claude Code JSONL session files — 6,367 session files parsed for per-message token usage and model breakdowns
  • Git history — 6,463 commits analyzed for code evolution, file growth, and contributor patterns
  • Pi session files — Alternative harness sessions parsed for cross-harness usage comparison

Caveats & Limitations

  • Tracking began mid-project; early development (pre-Feb 2025) and work on other machines are not captured
  • Subscription plans (Claude Pro/Team, Kimi Pro, etc.) make actual spend dramatically lower than PPU estimates
  • Some sessions used local models or free tiers not tracked in cost_events
  • Cache savings are estimated based on provider pricing differentials; actual savings vary by TTL and usage patterns

TLDR — Structured Code Summaries

Replaces full file reads with 500–1,200 token structured summaries. Intended to slash context size when agents read large source files.

4,340

Not working: 93% bypass rate (4,340 bypasses vs 341 interceptions) across all workspaces. TLDR is almost never triggering. Either file-size thresholds are too high, the daemon is not running, or the read-enforcer is missing hooks. Cost impact: effectively zero.

! Summary & Recommendations

TLDR Needs Fixing

93% bypass rate means TLDR is effectively dead. Investigate daemon health, hook wiring, and file-size thresholds. This is the highest-potential fix.

Add Telemetry

RTK and Caveman both lack pre/post token counts. Add a tokensSaved field to cost events or emit separate metrics events for each feature.

Cache Is Carrying Us

Cache reads (90.1B tokens) are doing almost all the heavy lifting. The $243K+ cache savings dwarfs anything these features could deliver. Don't lose sight of that.

Cost-Saving Features

What we built to spend less

PAN-636
Pi Harness Support
Added support for the Pi coding agent (multi-provider, cheaper models) as an alternative to Claude Code, enabling cost-effective model routing.
PAN-1249
Effect-Based Cost Tracking
Migrated cost tracking to Effect.js with proper error handling, enabling accurate per-issue, per-model, and per-pipeline-stage spend analysis.
PAN-1205
Cost Events Database
SQLite-backed cost_events table with deduplication, daily trends, model rollups, and agent-level cost attribution for fine-grained spend visibility.
Built-in
Prompt Caching (Automatic)
All major providers (Anthropic, OpenAI, Google) automatically cache prompts. Panopticon's long-context sessions naturally benefit from 5-minute and 1-hour cache TTLs.
Built-in
Model Routing (Cloister)
Cloister automatically routes work agents to the most cost-effective model for the task — Sonnet for bulk work, Opus for complex architecture, Haiku for quick tasks.
Built-in
vBRIEF Planning
Structured planning with vBRIEF specs reduces implementation iteration. Better plans = fewer agent restarts = fewer tokens wasted on wrong approaches.
Built-in
Workspace Isolation
Git worktree-based workspaces prevent cross-contamination. When an agent goes off the rails, only one workspace is affected — not the entire project history.
