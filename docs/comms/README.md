# Comms — user-facing content

Working home for content destined for the Overdeck landing page. Drafted here now; will be
organized into the site later.

- **`pr-faqs/`** — PR-FAQs (Amazon "working-backwards" press release + FAQ), one per feature or
  release. Written in the launched voice; availability notes stay honest about what's opt-in.
- **`blog/`** — longer-form blog articles and deep dives.

Convention: date-prefix filenames (`YYYY-MM-DD-slug.md`) and keep a short frontmatter block
(`title`, `date`, `audience`, `status`) so the eventual site tooling can index them.
