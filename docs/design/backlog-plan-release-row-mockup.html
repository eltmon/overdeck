Backlog item detail — the Plan → Release pickup gate

A new row in the backlog item's detail panel with two sequential operator gates on one line: Plan (Auto or Interactive) produces the spec + beads, then a Release control appears on the same row to let the item be picked up. This is a separate gate from Hold (park) and separate from any automatic planning — the operator drives plan, reviews it, then deliberately releases. Three independent gates

1
Plan — Auto or Interactive
Operator clicks Plan and picks the mode. Auto runs pan plan --auto --auto-start-off end-to-end; Interactive opens a planning session with Q&A. Produces the vBRIEF spec + beads → state becomes Planned.
2
Release — appears once Planned
On the same row, a Release control appears. Until the operator clicks it, a planned item is not pickable — this is the review checkpoint. Release → eligible for pickup by the flywheel / a work agent.
3
Hold — separate gate
An orthogonal park/pause. Independent of Plan and Release: you can hold an item at any stage and it will not be picked up, without losing its plan or release state.
Needs plan→ ⟳ Planning→ Planned (held)→ Released→ Picked up

How this differs from today

TodayAn item is auto-pickable the moment it is ready AND planned — isAutoPickable = ready && planned && !parked && !vetoed && !inPipeline. Having a plan = it can be grabbed. No review beat between "a plan exists" and "go work it."
ProposedInsert an explicit Release gate between Planned and pickable. Planned alone no longer makes it pickable; the operator reviews the plan, then Releases. Plan is now an explicit one-click action (Auto/Interactive) right on the detail row, not buried in a menu and not only an automatic flywheel step.
