Command Deck · Session view = AgentPane

Mockup #1 of the contextual-view restoration. Vocabulary follows pan-1549-workspace-panes.html (Stage → PaneBar → Pane). This is what a single-click on a session node (work / review / planning / reviewer) should open. Draft for alignment — no code yet.

Core move: clicking a session in the Navigator opens an AgentPane (paneType=agent) backed by that session. AgentPane already renders SessionPanel when handed a { session } — it's simply never handed one today (resolveAgentPane resolves only conversations). Fix = resolve sessions too + route single-click here. This also de-fangs the double-click bug (single-click stops yanking the layout).
