PAN-1549 · Workspace Panes

Layout + naming proposal — replacing the Zone* spatial names with a semantic Stage → Pane model. Reference design: the Subspace screenshots. Draft for discussion.

1 · Top-down vocabulary

Command Deck // shell — keep the name ├── Navigator NEW name (was Zone B — projects ▸ workspaces ▸ convos) └── Stage NEW name (was Zone C — the main work area) ├── PaneBar NEW // the persistent tab strip │ └── PaneTab // one tab, backed by a Pane └── Pane // active content, keyed by paneType ├── HomePane (was ZoneCOverview) │ ├── WorkspaceHeader (absorbs Zone A issue header) │ ├── Launcher (was "Command Launcher" — the omnibox) │ ├── AgentDock // agent pill launchers │ ├── ActionDock // Terminal / Files / Web / Commits │ └── Timeline // conversation history ├── AgentPane (was ZoneCConversation) ├── TerminalPane · FilesPane · CommitsPane · BrowserPane ├── PlanPane // folds in the vBRIEF + Beads overview tabs └── DocsPane // folds in the PRD / STATE / INFERENCE markdown tabs

The phrase to retire is ZoneCOverviewTabs: today it's "a strip of 10 sub-tabs." In the new model those tab bodies are re-homed — most into HomePane sections, the rest into PlanPane / DocsPane.
