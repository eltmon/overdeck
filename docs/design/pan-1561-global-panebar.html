PAN-1561 · Global PaneBar

The bigger step beyond PAN-1549: the PaneBar stops being a strip inside the Command Deck's Stage and becomes the outermost work surface. Tabs span all workspaces and projects in one flat, global strip — exactly the Subspace model. The Command Deck host view (project-tree + per-workspace Stage) is dissolved. Draft for discussion.

1 · The inversion — workspace ▸ panes becomes panes ▸ workspace

Before — PAN-1549 (nested, 3 levels deep)App └── Sidebar nav // Home · Command Deck · Board · … └── Command Deck view ├── Navigator // project ▸ workspace SELECT └── Stage (one workspace) ├── PaneBar // tabs of THIS workspace only └── Pane panesByWorkspace: Record<WsId, Pane[]> activePaneByWorkspace: Record<WsId, PaneId>
After — PAN-1561 (flat, outermost)App ├── PaneBar (GLOBAL, outermost) │ ├── PaneTab ws=PAN-1549 · agent │ ├── PaneTab ws=PAN-1547 · terminal │ ├── PaneTab ws=MYN-204 · docs │ └── Pane // the one active pane └── App Rail // app views only Command Deck host view (dissolved) Navigator as a container (gone) panes: Pane[] // each carries workspaceId activePaneId: PaneId | null

The project tree doesn't vanish — it demotes from a container that swaps the pane set into a pane-opener (the workspace switcher ▾ at the bar's left, or a HOME launcher). Selecting a workspace opens / focuses a tab; it never wipes the other open tabs.

2 · The surface, annotated

Colors: Global PaneBar · App Rail · Active Pane · HomePane regions. Each tab's dot + top-accent encodes which workspace it belongs to.
