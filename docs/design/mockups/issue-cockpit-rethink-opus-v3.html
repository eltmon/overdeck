What changed in v3 — and why (per your three notes)

  • Convoy nesting restored. Review is an expandable parent; its reviewer children (Correctness · Security · Performance · Requirements · Synthesis) nest under it with a verdict glyph (✓ approved / ✕ changes-requested) — matching FeatureItem.tsx ReviewGroup + the roundMetadata verdicts.
  • Select an agent → its conversation. Screen B: clicking a row swaps the center to SessionPanel — branch chip · Conversation/Terminal toggle · Tools, the transcript (assistant markdown, collapsible tool-call groups, user bubbles, working indicator), and the composer (model · effort · mic · context meter · send). Exactly the surface in your screenshot.
  • Header colors now follow the law (PAN-1148). Stepper: done = emerald, active = blue, blocked = red, ahead = neutral track. Gates: one dot shape — neutral = pending, red = failing, emerald = passing. Cost is the only cyan. No more dark-vs-grey "pending" mismatch, no triple-red noise.
  • Detail tabs thought through, not chips. The 15 flat tabs regroup by what they actually render into Quality (Review findings + Verification gates + Specialists convoy + Test/merge), Code (PR + CI checks + files + diff), Plan (vBRIEF List/DAG/Raw + PRD/STATE), Timeline (Activity + History), Discussion, Costs. Quality is shown fully populated above.
