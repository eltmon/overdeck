Big-bang revamp

One language for everything Panopticon shows you.

Today the Board, Command Deck, Agents page, and conversation views speak slightly different dialects — different vocab, different priorities, different mental models. This pass picks one vocabulary (pipeline phases · agent verbs · priority heat-map · one signal per element) and uses it everywhere. The result: five surfaces, one row primitive, one drawer.

01 · Information architecture

Sidebar collapses to four lenses on the same data.

Pipeline is the new default landing — answers "what's happening right now?" The other three operations surfaces are lenses: Board for tactile management, Command Deck for project drill-down, Agents for fleet observability. Everything else is observability or system.

What changed

Pipeline is new and takes the default-landing slot. It's the cross-project rollup grouped by lifecycle phase (Ship · Review · Work · Plan · Todo) — answers "what's machine-working, what needs me, what's in flight" in one glance.

Command Deck stays — your favorite — but is reframed as the per-project drill-down with the project tree on the left. Same row primitive as Pipeline, just filtered to one project + branch info.

Board is preserved but redesigned. The configuration controls (Harness, Agent model, Start) that today live on every card are pulled into Issue Detail, freeing cards to be dense status only.

Agents flips perspective from issue-centric to agent-centric. Each running agent is a card with its live stream excerpt and a back-link to its issue.

Awaiting Merge goes away as a standalone view. READY TO MERGE becomes a badge that surfaces in Pipeline and Board — no separate destination needed.

What's preserved

The two things you said work today: the project tree in Command Deck and the swarm rollup that nests review specialists under their issue. Both move into the new design unchanged.

03 · Shared primitives

Three components do all the heavy lifting.

Every surface composes the same atoms. Build them once, fix them once, change them once.

Issue Row

used in Pipeline · Command Deck · search results

Single-line summary of an issue and the agent acting on it. Priority left border, ID, status glyph, title, agent meta, runtime, avatar.

Every entry point lands on the same drawer.

No matter which surface you start from, clicking an issue opens the same Issue Detail drawer with the same state. You're never "stuck" in one view.

05 · Color discipline

Color always means the same thing.

One signal per element. Labels are taxonomy, never status. Phase color, priority color, verb color — each maps to a single token and never decorates.

--destructive Urgent priority · Stuck agents · Failed gates Action required — something is broken or critical.
--warning High priority · In Review column top border · REVIEW verb · Awaiting human A human needs to do something.
--info / --primary In Progress column · WORK verb · Machine-working metric A machine is actively doing something.
--signal-review Ship column · SHIP / PLANNING verbs · Project mark for Panopticon Specialist or planning activity — review pipeline state.
--success Done column · MERGED / READY TO MERGE badges · Passing gates Work complete or verified.
--signal-cost Spend metrics · Cost figures · Issue Detail drawer accent Money. Always cyan, always tabular numerals.
--muted-foreground Labels · Backlog / Todo column · Idle agents · Medium priority Neutral — no live signal. The "rest" state.
06 · What ships

Migration is non-trivial but bounded.

Every existing route has a one-to-one mapping into the new IA. The shared primitives can be built first (one PR), then each surface migrates independently. Today's kanban code becomes the Board surface; today's Command Deck conversation panel keeps its content but adopts the new shell; today's Agents page is replaced wholesale; Awaiting Merge is retired.

Build order (suggested)
  1. Shared tokens + primitives (Issue Row, Issue Card, verb badges, phase header)
  2. Pipeline (greenfield — uses only primitives)
  3. Issue Detail drawer (referenced by everything)
  4. Board (replaces kanban — same data model, new chrome)
  5. Command Deck refresh (keep tree, adopt new row component)
  6. Agents (wholesale replacement)
  7. Retire Awaiting Merge route
Risks worth naming
  • Drag-and-drop in Board needs a new state-machine for "moved card → phase transition"
  • Issue Detail must subscribe to multiple RPC streams (agent stream, activity feed, beads, verification) — design the subscription contract first
  • The drawer pattern can leak focus / scroll state — use a single store-owned drawer instance, not per-surface
  • Today's Cloister Deacon card on the Agents page needs a new home (probably Health)
