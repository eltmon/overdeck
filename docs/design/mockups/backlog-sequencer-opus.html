How to read this. Rank flows top-to-bottom; genuinely-independent issues share a tier. Node size is the AI's effort estimate, the left bar its impact tier, and the glowing ring means the issue is already moving through the pipeline (color = the phase it's in). A PRD chip means a .pan/drafts/<issue>.md exists; ✓ READY means a vBRIEF spec and beads are on disk, so a work agent could pick it up right now. Every rank carries the AI's reasoning, preserved in .pan/backlog/sequence.md across runs unless a delta justifies a change — click any node to read it. Three passes keep it honest: a one-time Creation pass builds the file, cheap Incremental passes touch only new / updated / closed issues, and a periodic Full review pass re-evaluates the whole backlog from scratch to catch drift. The AI also assesses each issue's condition: an amber ⚠ REFINE means the issue is too poorly specified to act on (a human must fix it), and a dimmed ⊘ STALE means the AI thinks it is probably no longer relevant and a candidate to close.  Mockup for PAN-1866; renders with ReactFlow 11 + @dagrejs/dagre in production (same stack as the vBRIEF DAG).
