Implementation Notes From Code Investigation

  • Existing data: /api/workspaces/:issueId already returns containers, stackHealth, frontendUrl, apiUrl, and pendingOperation.
  • Current gap: AwaitingMergePage posts to /api/workspaces/:issueId/rebuild-stack, then only shows a toast: “Watch the activity feed; the UAT link works once it is healthy.”
  • Issue tree: FeatureItem already expands a resources group and can render container nodes; the mockup adds a first-class “UAT environment” row with aggregate health.
  • Slide-out: the current WorkspaceCard compresses containers to running/total. Replace that with a compact service list and recent rebuild activity.
  • Polling: while a rebuild is pending or stack health is unhealthy, poll workspace state every 2-5 seconds; return to the normal 30-second interval once healthy.
