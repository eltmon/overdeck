What changed & how it maps to the code

  • Beads leaves the tab strip. Remove { id: 'beads', label: 'Beads' } from TABS and the activeTab === 'beads' branch in IssueMissionControl.tsx (shown struck-through above).
  • New third grid column. .missionBody in cockpitBody.module.css goes from minmax(280px,344px) minmax(0,1fr) to add a ~332px beads rail. The rail reuses BeadsTasksPanel's data hook (GET /api/issues/:id/beads).
  • "Working now" is real data, not invented. The beads endpoint already returns bd status = open | in_progress | closed plus startedAt. The in_progress bead is the highlight — no new backend needed (the frontend BeadTask type just needs 'in_progress' added to its union).

Color discipline (per design/style-guide/STYLE-GUIDE.md)

  • Blue = a machine is working → the active bead, the pulse. Not green.
  • Emerald = outcome/done → completed beads.
  • Neutral = rest state → upcoming beads, difficulty labels (taxonomy stays neutral).
  • One colored signal per row · DM Sans 500 · mono only for IDs/branch/model · rounded-sm badges, no pills.
