The scale story. overdeck has 522 open issues. Rendering all of them as a DAG would be unreadable — and the underlying .pan/backlog/sequence.md would balloon to ~120k tokens if every issue carried a full rationale. So the Sequencer keeps a one-line rationale for every issue and a full paragraph only for the active tier (≈ top 80), holding the file near ~50–65k tokens. The page mirrors that: a tiered view renders only the tier you select (Now / Next / Later / Someday) plus its dependency neighbors, the long tail collapses behind counts, and the segment pills + filters let you pivot the whole 522 by importance, condition, in-pipeline, or ready. A full Creation or Review pass reads ~370k tokens of issue bodies, so it runs as a chunked map-reduce; the cheap Incremental pass (default) only re-reads what changed. In-pipeline issues are pinned — never re-ranked under you. A ⊘ STALE issue the AI thinks is no longer relevant sinks to Someday and shows up under the ⊘ Stale candidates filter for one-click closing — unless it is already in the pipeline (like PAN-1803 here at #14), in which case it stays pinned in place and flagged so you can abort dead work instead of finishing it. Each issue also carries an operator pickup gate (Auto / 📌 Ready / ⛔ Block, see the rationale panel): Ready promotes it for auto-pickup, Block holds it out and sets a parked label — and when auto_pickup_backlog is on, the Flywheel pulls the highest-ranked eligible issue straight from this sequence instead of oldest-first — and with a sequence present the Sequencer is the single ranker, the Flywheel the executor (its only write is emergency-promoting a new pipeline-critical issue).

Three orthogonal per-issue axes — this retires needs-design/needs-discussion as a separate concept (click any node, see the panel): Planning {Skip / Auto / ⚑ Interactive} = how to plan it (Interactive is a hard human gate → surfaced as needs-you, never auto-run; AI defaults: AI-created or tiny → non-interactive, big features → interactive); Pickup gate {Auto / ✓ Ready / ⛔ Block} = whether the Flywheel may pick it; Condition {ok / ⚠ refine / ⊘ stale} = the state of the issue itself. The two parked labels decompose across these: needs-human-design → Planning Interactive, too-vague → Condition refine, on-hold → Pickup Block. Render budget: each tier shows the pinned set + that tier as a DAG up to ~150 nodes; larger tiers (Someday) render as a virtualized ranked list.  Mockup v2 for PAN-1866 — same ReactFlow 11 + @dagrejs/dagre stack as the vBRIEF DAG.
