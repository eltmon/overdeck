PAN-1737 v3 — UAT batch trains: always one batch ready, generations build forever

The model: every feature that passes review joins the queue. An assembly agent builds UAT batches — throwaway branches off main containing as many ready features as possible, in merge order, with cross-feature conflicts resolved inside the batch. When a new feature becomes ready, the next generation (brass-donkey ⊃ sea-monkey) assembles in the background while the current one stays fully testable — testing is never put on hold. Generations accumulate; you can UAT an older batch if the newest isn't what you want. Merging = promoting the batch you tested — main gets exactly the tree you exercised, conflict resolutions included. Merging a single feature out-of-band still works, but it invalidates the live batches and reassembly starts automatically.
